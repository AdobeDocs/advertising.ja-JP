# Adobe Advertising Universal ID要件ポリシー

最終更新日：2024年2月29日（PT）

このユニバーサル ID要件ポリシー（「ポリシー」）は、[!DNL Adobe's] On-demand Services for Adobe Advertising （「サービス」）を介して配信される広告に関連するユニバーサル IDの使用要件を規定しています。 このポリシーは、オンデマンドサービスを使用して、[!DNL Adobe]とお客様（「お客様」）との間の契約に参照によって組み込まれます。 本ポリシーで使用される大文字の用語は、本契約で定義されていない場合、本契約に記載されている意味を有します。

[!DNL Adobe]は、広告または広告がリダイレクトまたはリンクするウェブサイトが本ポリシーの要件に違反しているかどうかを単独の判断で判断する権利を留保します。この場合、イベント [!DNL Adobe]は当該広告を拒否し、その上で本サービスへのアクセスを終了または停止することができます。

以下および本契約に定める要件に加えて、お客様は、お客様がエンゲージすることを選択した各ユニバーサル ID プロバイダーが課す要件に加え、お客様の広告を配信するために使用されるサプライサイドプラットフォーム（SSP）またはその他のプラットフォームが課す要件に準拠する必要があります。 広告を配信するメディア企業やプラットフォームのポリシーを理解し、それに準拠するのは、お客様の責任です。

## 顧客データ要件

### 禁止データ

本サービスを通じて提供されるユニバーサル IDを使用することを選択することにより、お客様は、本オンデマンドサービスを通じてお客様が提供するすべてのクライアントデータが、以下のユニバーサル IDの仕様および本ポリシーのその他の要件に準拠していることを確認することに同意したものとみなされます。この要件は、[!DNL Adobe]によって随時修正される場合があります。

* 顧客データは、お客様がユーザーから直接収集するか、他の当事者からライセンスを取得したものであり、（i）顧客データが関連するユーザーから直接クライアントデータを収集したか、（ii）顧客からデータを収集した他の当事者からデータを取得した仲介者を介して顧客データを取得しました。

* クライアントデータは、web スクレイピングや、データの元のコレクションから切断されたソースからのデータベースの購入を介して発生していません。

* お客様は、本契約に定める目的のために、クライアントデータを[!DNL Adobe]およびそのパートナーに開示するために必要なすべての通知を提供し、すべての同意を得ました。

* 顧客データは、（i）お客様がターゲットオーディエンス向けのデータの作成または使用に関して直接または間接的に遵守することに同意したオプション（「オプトアウト」）、（ii）お客様が未成年であると知っている個人（適用される法域において法律で定義されている成年に満たない個人を指します）、または（iii）お客様が特定の健康状態に苦しんでいることを実際に知把握している個人に関連するものではありません。

* 本契約の発効日に従ってオプトアウトを受け取った場合、そのオプトアウトしたユーザーのユニバーサル IDは使用されなくなります。

### UID2禁止国

本サービスを通じてUID2 ユニバーサル IDを使用することを選択することにより、お客様は、UID2 ユニバーサル IDに関連して本オンデマンドサービスを通じて提供されるすべてのクライアントデータが、ここに記載されている[UID2 Prohibited Countriesのいずれかに居住する個人に関連していないことを確認することに同意したものとみなされます。](#prohibited-countries-uid2)

## 差別なし

お客様は、本サービスを使用して法律に違反して差別を行うことはできません。例えば、特定のグループの人々を誤って広告の対象としたり、特定のグループの人々を誤って広告を見ることを除外したり、広告に差別的なコンテンツを取り上げたりします。 お客様は、保護されたクラスに対する差別を避けるために本サービスを責任を持って使用する責任があります。これには、お客様の広告が保護されたクラスに異なる影響を与えないようにすることが含まれますが、これらに限定されません。

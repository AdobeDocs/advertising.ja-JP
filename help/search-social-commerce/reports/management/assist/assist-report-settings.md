# レポート設定の支援

*検索、ソーシャル、Commerceのクリックのトラッキングと、Adobe Advertising、Adobe Analyticsからのコンバージョントラッキング（[!DNL Analytics] 統合を使用）またはフィードでトークン（`ef_id`）のみを使用して提供される広告主*

| タブ | パラメーター | 説明 |
|----|----|----|
| 該当なし | [!UICONTROL Name] | （オプション）レポートおよびテンプレートの名前（レポートをテンプレートとして保存する場合）。 既存のテンプレートを適用する場合、デフォルトではテンプレート名が入力されます。 テンプレートを適用しない場合や名前を入力しない場合、レポートの名前はデフォルトで `<client name>-<date and time>-<report type>` になります（「acme - Apr 3, 2009 11:25:19 AM PDT - Keyword」など）。<br><br> オプションでカスタム名を入力できますが、ファイル拡張子は使用しません。<br><br> テンプレートを作成して [FTP ディレクトリにレポートを送信 &#x200B;](/help/search-social-commerce/reports/automation/ftp-reports.md) する場合は、ファイル名の任意の場所に「CSV」（大文字）を含めて、デフォルトの TSV 形式ではなく CSV 形式でファイルを作成できます。 [FTP ディレクトリに送信されるレポートのファイル名要件 &#x200B;](/help/search-social-commerce/reports/automation/ftp-reports.md) を参照してください。 |
|  | [!UICONTROL Save as template] | （スケジュールに従ってレポートを実行する場合を除き、オプション）レポート設定をテンプレートとして保存します。このテンプレートは、[!UICONTROL Reports] > [!UICONTROL Report Templates] ビューにあり、新しいレポートの作成に再利用できます。 レポートをテンプレートとして保存するには、チェック ボックスをオンにします。<br><br> スケジュールに従ってレポートを実行するには、設定をテンプレートとして保存する必要があります。<br><br><b> メモ：</b> 既存のテンプレートに基づいている場合でも、現在のパラメーターセットを新しいテンプレートとして保存できます。 |
|  | [!UICONTROL Type] | 生成するレポートタイプ。 |
| [!UICONTROL Basic  Settings] | [!UICONTROL Template] | （任意）適用するレポートテンプレート。テンプレートに従ってレポートオプションが事前設定されます。 レポートタイプ用に保存され、自身が使用できるすべてのテンプレートが一覧表示されます。<br><br> テンプレートを選択した場合でも、レポートオプションを変更できます。また、レポートを新しいテンプレートとして保存することもできます。 |
|  | [!UICONTROL Date Range] | データを生成する日付範囲：<ul><li><i>[ プリセット範囲 ]:</i> 一般的な時間増分のリストで、<i>[!UICONTROL Today]</i> から <i>[!UICONTROL Last 180 Days]</i> までの範囲です。 デフォルトは <i>[!UICONTROL Last 7 Days]</i> で、データを使用できる過去 7 日間がレポートされます。 <b> メモ：</b> <i>[!UICONTROL Last Month]</i>、<i>[!UICONTROL Last 3 Months]</i>、<i>[!UICONTROL Last 6 Months]</i> に、前のカレンダー月のデータが表示されます。</li><li><i>[!UICONTROL Custom Date Range]:</i> 開始日と終了日を指定します。 MM/DD/YYYY または M/D/YYYY の形式で日付を入力するか、フィールドの横にある ![&#x200B; カレンダー &#x200B;](/help/search-social-commerce/assets/calendar.png " カレンダー ") をクリックして日付を選択します。</li></ul> |
|  | [!UICONTROL Filter By] | （[!UICONTROL Campaign Assist Report] のみ）特定のポートフォリオのデータについてレポートするか、特定の広告ネットワークのデータについてレポートするか。<ul><li><i>[!UICONTROL Portfolio]</i> （デフォルト）：キャンペーンのデータを 1 つ以上のポートフォリオに含めます。</li><li><i>[!UICONTROL Search Engine]:</i> レポートタイプの場合に適用されるように、1 つ以上の広告ネットワークにキャンペーンのデータを含める。 |
|  | \[プライマリ フィルタ\] | （[!UICONTROL Campaign Assist Report] のみ）含めるデータコンポーネント。 選択しない場合、該当する広告ネットワークで発生したコンバージョンパスのイベントタイプの各パターンに対するデータ行がレポートに含まれます。 オプションで、レポートされるデータをフィルタリングし、指定したコンポーネントおよびサブコンポーネントで一連のイベントが少なくとも 1 回発生した場合にのみ行が含まれるようにすることもできます。 ポートフォリオでフィルタリングするか広告ネットワークでフィルタリングするかに応じて、含めるコンポーネントを指定します。<ul><li><i>[!UICONTROL Portfolio]:</i> 1 つ以上のポートフォリオまたはそのサブコンポーネント（キャンペーンまたは広告グループ）。 コンポーネントとそのすべてのサブコンポーネントを選択するには、コンポーネント名の横にあるチェックボックスを選択します。 サブコンポーネントを選択するには、サブコンポーネント名の横にあるチェックボックスをオンにし、>> をクリックして [!UICONTROL Selected Filters] 列に移動します。 例えば、Portfolio 1 とそのすべてのキャンペーンおよび広告グループのデータを取得するには、Portfolio 1 の横にあるチェックボックスをオンにします。 Portfolio 1 のキャンペーン 1 で 1 つ以上のイベントが発生した際にデータを取得するには、「Portfolio 1」を展開して、キャンペーン 1 の横にあるチェックボックスのみをオンにします。</li><li><i>[!UICONTROL Search Engine]:</i> 1 つ以上の広告ネットワークまたはそのサブコンポーネント（アカウント、キャンペーン、広告グループ）。 コンポーネントとそのすべてのサブコンポーネントを選択するには、コンポーネント名の横にあるチェックボックスをオンにし、>> をクリックして [!UICONTROL Selected Filters] 列に移動します。 サブコンポーネントを選択するには、サブコンポーネント名の横にあるチェックボックスをオンにし、>> をクリックして [!UICONTROL Selected Filters] 列に移動します。 例えば、[!DNL Google Ads] アカウント、キャンペーン、広告グループで 1 つ以上のイベントが発生した際にデータを取得するには、[!DNL Google AdWords] の横にあるチェックボックスをオンにします。 [!DNL Google Ads] アカウント 1 のキャンペーン 1 のデータのみを取得するには、「[!DNL Google Ads]」、「アカウント 1」の順に展開し、キャンペーン 1 の横にあるチェックボックスのみを選択します。</li></ul><b> 注：</b><ul><li>リスト内のコンポーネントを展開するには（広告ネットワーク上のアカウントを一覧表示する場合など）、コンポーネントの横にある ![&#x200B; 右矢印アイコン &#x200B;](/help/search-social-commerce/assets/compressed-item.png " 右矢印アイコン ") をクリックします。</li><li>項目のコンポーネントの種類を確認するには、その上にカーソルを置きます。</li><li>デフォルトでは、a） アクティブで最適化されたポートフォリオとそのアクティブなコンポーネント、b） アクティブで有効な広告ネットワークアカウント、キャンペーン、およびそのアクティブなコンポーネントのみが表示されます。 一時停止または削除されたコンポーネントを表示するには、**[!UICONTROL Show]** の横の ![&#x200B; 下矢印 &#x200B;](/help/search-social-commerce/assets/arrow-down-expand.png " 下矢印 ") をクリックし、「**[!UICONTROL All]**」を選択します。</li><li>ポートフォリオ別にアシストレポートを生成すると、指定したポートフォリオに現在マッピングされているキャンペーンのデータが得られます。 レポートには、日付範囲内にポートフォリオにあったが、まだ存在しないキャンペーンのデータは含まれていません。</li></ul> |
| [!UICONTROL Columns] | [!UICONTROL Use revenue and derived metrics from] | 次のデフォルト・ビューのセットのいずれかに共通するすべての収益およびカスタム（導出）メトリック列がレポートに移入されます（つまり、最小共通分母を使用します）。<ul><li><i> すべて：</i> 広告主の製品設定に適用されるように、[!UICONTROL Portfolios] ビューと [!UICONTROL Campaigns] ビュー内のすべてのタブについて、デフォルトビューで共有される列のスーパーセットを含めます。</li><li><i>[!UICONTROL Portfolios]:</i> デフォルト・ビューで共有される列を、[!UICONTROL Portfolios] のビュー内のすべてのタブに含める。</li><li><i>[!UICONTROL Search]:</i> デフォルト・ビューで共有される列を、[!UICONTROL Campaigns] のビュー内のすべてのタブに含める。</li><li><i>[!UICONTROL Display]:</i> 古い</li><li><i>[!UICONTROL Social]:</i> 古い</li><ul><b> 注：</b><ul><li>使用できる検索トラフィック指標（クリック数やインプレッション数など）がなく、検索トラフィック指標を含む派生指標は使用できません。</li><li>オプションで、使用可能な売上高または派生指標列を削除または並べ替えることができます。</li><li>プロパティ列はレポート用に事前定義されていますが、列を追加することもできます。</li></ul> |
|  | [!UICONTROL Path Size] | パターン内の項目（イベントタイプ、キーワードまたはプレースメント、広告グループ、キャンペーン）の最小数と最大数。 デフォルトのパスサイズは、1 つ以上の項目および最大 5 つの項目です。 オプションで、2 つ以上の項目を含むパスのみを表示できます。 項目のタイプと最大数は、レポートによって異なります。<ul><li>[!UICONTROL Channel Assist Report]：広告主の [&#x200B; クリックルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d) および [&#x200B; インプレッションルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j) 内で発生したコンバージョンパスの最も古いイベント N 個までのデータを項目化します。 例えば、1 つ以上 5 つまでのパスサイズを選択した場合、レポートには最大 5 つのイベントを含み、追跡されるイベントタイプ（「検索クリック」や「インプレッションの表示」など）のパターンごとに 1 行のコンバージョンパスが含まれます。 パスには、最大 30 個のイベントを含めることができます。</li><li>[!UICONTROL Keyword Assist Report]：広告主の [&#x200B; クリックルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j) および [&#x200B; インプレッションルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d) 内で発生したコンバージョンパスの最も古い検索キーワードまたはプレースメントについて、最大 N 個のデータを項目化します。 例えば、1 つ以上 5 つまでのパスサイズを選択した場合、レポートには最大 5 つのキーワードまたはプレースメントを含むパスが含まれ、追跡されたキーワード文字列またはプレースメントのパターンごとに 1 行が含まれます。 パスには、最大 10 個のイベントを含めることができます。</li><li>[!UICONTROL Campaign Assist Report]：広告主の [&#x200B; クリックのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j) および [&#x200B; インプレッションのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d) 内で発生したコンバージョンパスの最も古い N 個までのキャンペーンのデータを項目化します。 例えば、1 つ以上 5 つまでのパスサイズを選択した場合、レポートには最大 5 つのキャンペーンを含むパスが含まれ、追跡されるキャンペーンのパターンごとに 1 行が含まれます。 パスには、最大 10 個のキャンペーンを含めることができます。</li></ul><b> 注：</b><ul><li>アシスト レポートのコンバージョン データを表示するには、適切なコンバージョン列を追加する必要があります。</li><li>コンバージョンパスに、レポートの項目別に説明した数よりも多くのイベントタイプ、キーワードまたはプレースメント、広告グループまたはキャンペーンが含まれている場合、レポートには、項目数が多いことで生じたコンバージョンのデータを要約した追加の行（10 を超えるイベントで生じたコンバージョンの 1 行など）が含まれます。</li><li>2 つ以上の項目を含むパスを含める場合、合計は、1 つ以上の項目を含むパスの合計よりも低くなる場合があります。</li></ul> |
| [!UICONTROL Columns] | \[ レポート列\] | レポートに表示されるデータの列とその順序：<ul><li>列を追加するには、左列の指標名をクリックし、![&#x200B; 右矢印 &#x200B;](/help/search-social-commerce/assets/arrow-right-customize.png " 右矢印 ") をクリックします。</li><li>列を削除するには、右列の指標名をクリックし、![&#x200B; プロパティ &#x200B;](/help/search-social-commerce/assets/property-columns.png " プロパティ ")：広告ネットワークアカウントまたはポートフォリオコンポーネントのプロパティ名と ID （[!UICONTROL Campaign Status] など）</li><li>![&#x200B; 広告主について追跡されたコンバージョン指標の売上高指標 &#x200B;](/help/search-social-commerce/assets/revenue-metric-columns.png " 売上高指標 ")Adobe Analyticsから同期されたコンバージョン指標やサイトエンゲージメント指標を含む）</li><li>広告主が作成したカスタム派生指標の ![&#x200B;

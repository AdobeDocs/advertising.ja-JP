# アカウントコンポーネントは、いつ在庫フィードによって作成または削除されますか？

*[!DNL Google Ads]、[!DNL Microsoft Advertising]、[!DNL Yahoo! Japan Ads] （削除アクションのみ）および [!DNL Yandex] アカウントのみ*

在庫フィードファイルがテンプレートによって伝播されると、次のように、アカウントコンポーネントが作成および削除されます。

>[!NOTE]
>
>広告グループ内で重複した広告が作成されることはありません。

| シナリオ | 例 | アクション |
|----|----|----|
| フィードデータには、キャンペーン名、広告グループ名、キーワードまたは製品グループで使用される列の新しい値が含まれます。 | 前のファイル：<br>Campaign=Hats<br>Campaign=Gloves<br><br> 新しいファイル：<br>Campaign=Shoes | 広告ネットワークに存在しない場合は、新しいキャンペーン、広告グループ、キーワードまたは製品グループが作成されます。 |
| フィードデータに、広告で使用される列の新しい値が含まれています。 | 前のファイル：広告が含まれる価格=20<br><br> 新しいファイル：同じ広告に対して、価格=10 | 展開されたテキスト広告、[!DNL Yahoo! Japan ads] または [!DNL Yandex] 広告 [!DNL Microsoft Advertising] 広告コピーが変更されると、既存の広告は削除され、新しい広告が作成されます。<br><br> 他の広告タイプの広告コピーが変更されたとき、または該当する列が広告の [!DNL Google Ads] しい広告パラメーター（{param1} または {param2}）に使用されたとき、既存の広告が更新されます。 |
| キャンペーン、広告グループ、キーワードまたは製品グループのテンプレート設定が、前回の伝播以降に変更されました。 | 前の設定：Keyword=[Keyword]<br><br>New 設定：Keyword=&lt;Color>[Keyword] | 広告ネットワークに存在しない場合は、新しいキャンペーン、広告グループ、キーワードまたは製品グループが作成されます。 |
| 広告のテンプレート設定は、前回の伝播以降に変更されました。 | 以前の設定：広告 description=&quot;Buy [category] now.&quot;<br><br> 新しい設定：広告の説明=&quot;今すぐ [ ブランド ] を購入&quot; | 展開されたテキスト広告、[!DNL Yahoo! Japan ads] または [!DNL Yandex] 広告 [!DNL Microsoft Advertising] 広告コピーが変更されると、既存の広告は削除され、新しい広告が作成されます。<br><br> 他の広告タイプの広告コピーが変更されたとき、または広告の 1 つの [!DNL Google Ads] 広告パラメーター（{param1} または {param2}）に使用されている列が変更されたことを反映した場合、既存の広告が更新されます。 |
| 新しいフィードデータに、既存のキャンペーンまたは広告グループの行が含まれていません。 | 該当なし | 既存のキャンペーンと広告グループは元の状態のままです。 |
| 新しいフィードデータに、既存の広告グループ、広告、キーワードまたは製品グループの行が含まれていない。 | 該当なし | [ フィードデータの設定 ](feed-settings-manage.md#feed-data-settings) に従って、既存の広告グループ、広告、キーワードまたは製品グループは、そのまま残るか、一時停止するか、削除されます。 |
| 既存の親製品グループの新しいフィードデータに、既存の子製品グループの行が含まれていません。 | 該当なし | 既存の親製品グループは、[ フィードデータ設定 ](feed-settings-manage.md#feed-data-settings) に従って、そのまま残るか削除されます。 <b> メモ：</b> 不足している行項目を一時停止するようにフィードデータ設定が設定されている場合、製品グループを一時停止できないので、親製品グループは削除されたままになります。 |
| 新しいフィードデータに含まれる広告グループ、広告、キーワードまたは製品グループの行は、a）以前のデータでは除外されていたのに、b）それ以降はスキップされ、[ フィードデータの設定 ](feed-settings-manage.md#feed-data-settings) に従って一時停止されていました。 | 該当なし | 履歴や品質スコアを失うことなく、既存の広告グループ、広告、キーワードまたは製品グループが再アクティブ化されます。 |
| 新しいフィードデータに含まれる広告グループ、広告、キーワードまたは製品グループの行は、a）以前のデータでは含まれていたが、b）それ以降は省略され、[ フィードデータの設定 ](feed-settings-manage.md#feed-data-settings) に従って削除されました。 | 該当なし | 新しい広告グループ、広告、キーワードまたは製品グループが作成されます。 |
| 「[!UICONTROL Delete negative keywords when omitted from list]」に対するキャンペーンレベルまたは広告グループレベルのオプションを無効にしました。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br> 広告グループには、既に負のキーワード「SUV」が含まれています。 | リストにない既存のネガティブキーワードは、そのまま残ります。 |
| キャンペーンレベルまたは広告グループレベルのオプションを「[!UICONTROL Delete negative keywords when omitted from list]」に対して有効にしましたが、リストに含まれていない負のキーワードが存在します。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br> 広告グループには、既に負のキーワード「SUV」が含まれています。 | テンプレートを使用して以前に作成された未指定の負のキーワードは、フィードファイルがテンプレートを通じて伝播されると削除されます。 ただし、他の手段（プレーンバルクシート、[!UICONTROL Campaigns] ビュー、広告ネットワークの広告エディターなど）で作成された未指定の負のキーワードは、そのまま残ります。 | | 投稿されたフィード ファイルのコンポーネントに対してスケジュールされた終了日が発生します。 | 該当なし | 既存のキャンペーンは保持されます。 既存の広告グループ、広告、キーワードは、「フィードデータの設定 [ に従って、そのまま残ったり、一時停止したり、削除されたりし ](feed-settings-manage.md#feed-data-settings) す。 |
| 品目の在庫レベルが、[ フィードデータ設定 ](feed-settings-manage.md#feed-data-settings) で指定された最小値を下回ります。 | 前のファイル：stock=10<br><br> 新しいファイル：stock=0 | 既存のキャンペーンは保持されます。 既存の広告グループ、広告、キーワード、製品グループは、[ フィードデータの設定 ](feed-settings-manage.md#feed-data-settings) に従って、一時停止または削除されます。 |
| アイテムの在庫レベルが、「フィードデータ設定 [ で指定された最小値を超え ](feed-settings-manage.md#feed-data-settings) います。 | 前のファイル：stock=0<br><br> 新しいファイル：stock=10 | 既存の広告、キーワードまたは製品グループが一時停止すると、履歴や品質スコアを失うことなく再アクティブ化されます。 広告、キーワードまたは製品グループが存在しない場合（例えば、在庫レベルが最小値を下回っていたために以前に削除された場合）、新しい広告、キーワードまたは製品グループが作成されます。 |

# アカウントコンポーネントは、いつ在庫フィードによって作成または削除されますか？

*[!DNL Google Ads], [!DNL Microsoft Advertising], [!DNL Yahoo! Japan Ads] （削除アクションのみ）、 [!DNL Yandex] アカウントのみ*

在庫フィードファイルがテンプレートによって伝播されると、次のように、アカウントコンポーネントが作成および削除されます。

>[!NOTE]
>
>広告グループ内で重複した広告が作成されることはありません。

| シナリオ | 例 | アクション |
|----|----|----|
| フィードデータには、キャンペーン名、広告グループ名、キーワードまたは製品グループで使用される列の新しい値が含まれます。 | 以前のファイル：<br>キャンペーン=帽子<br>Campaign=手袋<br><br>新しいファイル：<br>Campaign=靴 | 広告ネットワークに存在しない場合は、新しいキャンペーン、広告グループ、キーワードまたは製品グループが作成されます。 |
| フィードデータに、広告で使用される列の新しい値が含まれています。 | 前のファイル：広告に含まれる価格= 20<br><br>新規ファイル：同じ広告に対して、price=10 | 広告がにコピーされたとき [!DNL Microsoft Advertising] 展開されたテキスト広告、 [!DNL Yahoo! Japan ads]、または [!DNL Yandex] 広告が変更されると、既存の広告は削除され、新しい広告が作成されます。<br><br>他の広告タイプの広告コピーが変更されたとき、または該当する列がに使用されたとき [!DNL Google Ads] 広告パラメーター（{param1} または {param2}）を選択した場合、既存の広告が更新されます。 |
| キャンペーン、広告グループ、キーワードまたは製品グループのテンプレート設定が、前回の伝播以降に変更されました。 | 前の設定：Keyword=[キーワード]<br><br>新しい設定：Keyword=&lt;color>[キーワード] | 広告ネットワークに存在しない場合は、新しいキャンペーン、広告グループ、キーワードまたは製品グループが作成されます。 |
| 広告のテンプレート設定は、前回の伝播以降に変更されました。 | 以前の設定：広告の説明=&quot;購入 [category] 今だ。」<br><br>新しい設定：広告 description=&quot;Buy [ブランド] 今だ。」 | 広告がにコピーされたとき [!DNL Microsoft Advertising] 展開されたテキスト広告、 [!DNL Yahoo! Japan ads]、または [!DNL Yandex] 広告が変更されると、既存の広告は削除され、新しい広告が作成されます。<br><br>他の広告タイプの広告コピーが変更されたとき、または変更が 1 つのに使用されている列の変更を反映したとき [!DNL Google Ads] 広告パラメーター（{param1} または {param2}）を選択した場合、既存の広告が更新されます。 |
| 新しいフィードデータに、既存のキャンペーンまたは広告グループの行が含まれていません。 | 該当なし | 既存のキャンペーンと広告グループは元の状態のままです。 |
| 新しいフィードデータに、既存の広告グループ、広告、キーワードまたは製品グループの行が含まれていない。 | 該当なし | 既存の広告グループ、広告、キーワードまたは製品グループは、 [フィードデータ設定](feed-settings-manage.md#feed-data-settings). |
| 既存の親製品グループの新しいフィードデータに、既存の子製品グループの行が含まれていません。 | 該当なし | 既存の親製品グループは、に従って、そのまま残るか削除されます [フィードデータ設定](feed-settings-manage.md#feed-data-settings). <b>注意：</b> 不足している行項目を一時停止するようにフィードデータ設定が設定されている場合、製品グループを一時停止できないので、親製品グループはまだ削除されます。 |
| 新しいフィードデータに含まれる広告グループ、広告、キーワードまたは製品グループの行が、（a）以前のデータでは省略されていたが、（b）それ以降に省略され、に従って一時停止されていた [フィードデータ設定](feed-settings-manage.md#feed-data-settings). | 該当なし | 履歴や品質スコアを失うことなく、既存の広告グループ、広告、キーワードまたは製品グループが再アクティブ化されます。 |
| 新しいフィードデータに含まれる広告グループ、広告、キーワードまたは製品グループの行が、（a）以前のデータではあったが、（b）それ以降は省略され、に従って削除されました [フィードデータ設定](feed-settings-manage.md#feed-data-settings). | 該当なし | 新しい広告グループ、広告、キーワードまたは製品グループが作成されます。 |
| &quot; &quot;に対するキャンペーンレベルまたは広告グループレベルのオプションを無効にしました[!UICONTROL Delete negative keywords when omitted from list].」と入力します。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br>広告グループには、既に負のキーワード「SUV」が含まれています。 | リストにない既存のネガティブキーワードは、そのまま残ります。 |
| 「」に対してキャンペーンレベルまたは広告グループレベルのオプションを有効にしました[!UICONTROL Delete negative keywords when omitted from list]リストにない、」および負のキーワードが存在します。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br>広告グループには、既に負のキーワード「SUV」が含まれています。 | テンプレートを使用して以前に作成された未指定の負のキーワードは、フィードファイルがテンプレートを通じて伝播されると削除されます。 ただし、他の手段で作成された不特定の負のキーワード（プレーンバルクシートなど） [!UICONTROL Campaigns] ビュー、または広告ネットワークの広告エディター内のビューは、そのまま残ります。 | | 投稿されたフィード ファイルのコンポーネントに対してスケジュールされた終了日が発生します。 | 該当なし | 既存のキャンペーンは保持されます。 既存の広告グループ、広告、キーワードは、そのまま残るか、一時停止するか、削除するかが、 [フィードデータ設定](feed-settings-manage.md#feed-data-settings). |
| 項目の在庫レベルが、 [フィードデータ設定](feed-settings-manage.md#feed-data-settings). | 前のファイル：stock=10<br><br>新しいファイル：stock=0 | 既存のキャンペーンは保持されます。 既存の広告グループ、広告、キーワード、製品グループは、 [フィードデータ設定](feed-settings-manage.md#feed-data-settings). |
| 項目の在庫レベルが、 [フィードデータ設定](feed-settings-manage.md#feed-data-settings). | 前のファイル：stock=0<br><br> 新規ファイル：stock=10 | 既存の広告、キーワードまたは製品グループが一時停止すると、履歴や品質スコアを失うことなく再アクティブ化されます。 広告、キーワードまたは製品グループが存在しない場合（例えば、在庫レベルが最小値を下回っていたために以前に削除された場合）、新しい広告、キーワードまたは製品グループが作成されます。 |

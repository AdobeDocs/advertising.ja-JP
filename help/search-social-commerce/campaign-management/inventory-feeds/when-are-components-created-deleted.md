# 在庫フィードでアカウントコンポーネントが作成または削除されるのはいつですか？

*[!DNL Google Ads]、[!DNL LY Ads] （削除操作のみ）、[!DNL Microsoft Advertising]、および[!DNL Yandex] アカウントのみ*

在庫フィードファイルがテンプレートを通じて伝播されると、アカウントコンポーネントが次のように作成および削除されます。

>[!NOTE]
>
>広告グループ内で重複する広告が作成されることはありません。

| シナリオ | 例 | アクション |
|----|----|----|
| フィードデータには、キャンペーン名、広告グループ名、キーワード、製品グループで使用される列の新しい値が含まれます。 | 前のファイル：<br>Campaign=Hats<br>Campaign=Gloves<br><br>新しいファイル：<br>Campaign=Shoes | 広告ネットワークに新しいキャンペーン、広告グループ、キーワード、製品グループが存在しない場合は、作成されます。 |
| フィードデータには、広告で使用される列の新しい値が含まれます。 | 前のファイル：広告に含まれるprice=20<br><br>新しいファイル：同じ広告の場合、price=10 | [!DNL LY Ads]、[!DNL Microsoft Advertising]拡張テキスト広告、または[!DNL Yandex]広告の広告コピーが変更されると、既存の広告が削除され、新しい広告が作成されます。<br><br>他の広告タイプの広告コピーが変更された場合、または該当する列が広告内の[!DNL Google Ads]広告パラメーター（{param1}または{param2}）に使用されている場合、既存の広告が更新されます。 |
| キャンペーン、広告グループ、キーワード、または製品グループのテンプレート設定は、前回の伝播以降に変更されました。 | 前の設定:Keyword=[ キーワード ]<br><br>新しい設定：キーワード=&lt;色>[ キーワード ] | 広告ネットワークに新しいキャンペーン、広告グループ、キーワード、製品グループが存在しない場合は、作成されます。 |
| 広告のテンプレート設定は、前回の伝播以降に変更されました。 | 前の設定：Ad description=&quot;今すぐ[ カテゴリ ]を購入する&quot;<br><br>新しい設定：Ad description=&quot;今すぐ[ ブランド ]を購入する&quot;。 | [!DNL LY Ads]、[!DNL Microsoft Advertising]拡張テキスト広告、または[!DNL Yandex]広告の広告コピーが変更されると、既存の広告が削除され、新しい広告が作成されます。<br><br>他の広告タイプの広告コピーが変更された場合、または変更が広告内の単一の[!DNL Google Ads]広告パラメーター（{param1}または{param2}）に使用されている列の変更を反映した場合、既存の広告が更新されます。 |
| 新しいフィード データには、既存のキャンペーンまたは広告グループの行が含まれていません。 | なし | 既存のキャンペーンと広告グループはそのまま残ります。 |
| 新しいフィード データには、既存の広告グループ、広告、キーワード、製品グループの行が含まれていません。 | なし | [&#x200B; フィードデータ設定](feed-settings-manage.md#feed-data-settings)に従って、既存の広告グループ、広告、キーワード、または製品グループがそのまま残るか、一時停止するか、削除されます。 |
| 既存の親製品グループの新しいフィード データには、既存の子製品グループの行が含まれていません。 | なし | [&#x200B; フィードデータ設定](feed-settings-manage.md#feed-data-settings)に従って、既存の親製品グループは現状のまま、または削除されます。 <b> メモ：</b>欠落している行アイテムを一時停止するようにフィード データ設定が構成されている場合、製品グループを一時停止できないため、親の製品グループは削除されます。 |
| 新しいフィード データには、以前のデータではa）だった広告グループ、広告、キーワード、または製品グループの行が含まれていますが、それ以降はb）が省略され、[&#x200B; フィード データ設定](feed-settings-manage.md#feed-data-settings)に従って一時停止されました。 | なし | 既存の広告グループ、広告、キーワード、製品グループは、履歴や品質スコアを失うことなく、再アクティブ化されます。 |
| 新しいフィード データには、以前のデータではa）でしたが、それ以降はb）が省略され、[&#x200B; フィード データ設定](feed-settings-manage.md#feed-data-settings)に従って削除された広告グループ、広告、キーワード、製品グループの行が含まれます。 | なし | 新しい広告グループ、広告、キーワード、製品グループが作成されます。 |
| キャンペーンまたは広告グループレベルのオプションを「[!UICONTROL Delete negative keywords when omitted from list]」に対して無効にしました。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br>広告グループには、すでにネガティブキーワード「SUV」が含まれています。 | リストに含まれていない既存の否定的なキーワードはそのまま残ります。 |
| キャンペーンまたは広告グループレベルのオプションを「[!UICONTROL Delete negative keywords when omitted from list]」に有効にしましたが、リストに含まれていない負のキーワードが存在します。 | ネガティブキーワードリストには、「クーペ」と「スポーツカー」が含まれます。<br><br>広告グループには、すでにネガティブキーワード「SUV」が含まれています。 | テンプレートを使用して以前に作成した未指定の負のキーワードは、フィードファイルがテンプレートを通じて伝播されると削除されます。 ただし、その他の手段（プレーンバルクシート、[!UICONTROL Campaigns] ビュー、広告ネットワークの広告エディターなど）を使用して作成された未指定のネガティブキーワードは、そのまま残ります。 |
| 投稿されたフィード ファイルのコンポーネントの予定終了日が発生します。 | なし | 既存のキャンペーンはそのまま残ります。 [&#x200B; フィードデータ設定](feed-settings-manage.md#feed-data-settings)に従って、既存の広告グループ、広告、キーワードをそのまま使用するか、一時停止するか、削除します。 |
| 品目の在庫レベルが、[&#x200B; フィード データ設定](feed-settings-manage.md#feed-data-settings)で指定された最小値を下回っています。 | 前のファイル：stock=10<br><br>新しいファイル：stock=0 | 既存のキャンペーンはそのまま残ります。 [&#x200B; フィードデータ設定](feed-settings-manage.md#feed-data-settings)によると、既存の広告グループ、広告、キーワード、製品グループは一時停止または削除されます。 |
| アイテムの在庫レベルが、[&#x200B; フィード データ設定](feed-settings-manage.md#feed-data-settings)で指定された最小値を超えています。 | 前のファイル：stock=0<br><br>新しいファイル：stock=10 | 既存の広告、キーワード、製品グループが一時停止されると、履歴や品質スコアを失うことなく、再アクティブ化されます。 広告、キーワード、または製品グループが存在しない場合（例えば、在庫レベルが最小値を下回ったために以前に削除された場合）、新しいグループが作成されます。 |

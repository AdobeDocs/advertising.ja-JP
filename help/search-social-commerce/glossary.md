# 用語集 {#glossary}

## A-B {#a-b}

**広告グループ:** キャンペーンの広告とそれに関連するキーワード、プレースメント、商品グループのセット。

**広告バリエーション:** 広告グループまたは広告戦略内の任意の広告。

**[AMO ID](/help/integrations/analytics/ids.md#amo-id):** Adobe Systems Advertising がキャンペーンに関するデータを Adobe Analytics と共有できるようにするトラッキングコード。 それは `s_kwcid=`から始まります。

**入札単位:** 入札が行われる単位のSearch、Social、およびコマース用語。

* CPC キャンペーンの場合、これは検索または内容 キャンペーンの場合はキーワードとそのマッチタイプ、ショッピング キャンペーンの場合はユニットレベルの商品グループ(最も低い分類レベル)、動的検索 広告キャンペーンの場合は動的検索 ターゲットです。 同じキーワードとマッチタイプの組み合わせ、同じ商品グループ、または同じ動的検索 ターゲットが 1 つのキャンペーン内の複数の広告グループ内で発生する場合、すべてのインスタンスは同じ入札単位と見なされ、同じ入札を持ちます。

* [!DNL Maximize Clicks]、[!DNL Maximize Conversion Value]、[!DNL Maximize Conversions]、[!DNL Target Cost Per Acquisition]、または[!DNL Target Return on Ad Spend]支出戦略を使用するキャンペーンの場合、各キャンペーンは入札単位です。

* キーワードを使用しない [!DNL Yahoo! Display Network] のキャンペーンでは、広告グループ内のすべての広告が同じ入札を持ち、同じ入札単位と見なされます。

**入札単位制約:** 「制約」を参照してください。

## C-D {#c-d}

**キャンペーン:** 予算、期間、ターゲティングなどの設定を共有する、1 つの広告 アカウント内の一連の広告グループ。 **注:** [!DNL Baidu] にはキャンペーンの概念はありませんが、Search、Social、コマースは、Search、Social、コマース内で同期される既存の [!DNL Baidu] アカウントの関連する広告グループのセットごとに疑似キャンペーンを作成します。

**大文字と小文字を区別するフィールド:** 大文字と小文字を区別するフィールドまたはクエリでは、大文字 (C など) とすべて小文字文字 (c など) の扱いが異なります。 例えば、Car は car とは異なる値として扱われます。

**click:** 1 回のユーザークリック、またはオンライン 広告内で行います。

**クリックルックバックウィンドウ:** 一連のイベント有料クリックが発生してから、そのクリックをコンバージョンに結び付けることができる日数を指定する広告主レベルの設定。

**クリック時間:** 一意の IP アドレスを持つエンド ユーザーが広告ネットワーク内の広告を最初にクリックする時間。

**クリックスルー率:** (CTR) クリック数を広告のインプレッション数で割った値。 検索クエリーとの関連性が最も高い広告のクリックスルー率が高くなります。

**click-トラッキング URL:** キーワード、広告バリエーションまたは配置でのクリックを追跡するコードが埋め込まれたトラッキング テンプレートまたは宛先URL。

**制約:** (ポートフォリオを持つ広告主、標準ポートフォリオの入札ユニットにのみ適用)特定のキーワードまたは広告に入札するためのルール。 ポートフォリオレベルの制限および推奨入札戦略に優先します。

**コンバージョン:** 通常、指標としてキャプチャされる広告ユーザークリックの終了後にアクションが完了したこと。 例としては、登録や購入などがあり、数や金額を表すことができます。 コンバージョンは 1 つ以上のトランザクションイベントで構成できますが、&quot;コンバージョン&quot; と &quot;トランザクション&quot; という用語は同じ意味で使用されることがよくあります。

**コンバージョン トラッキング:** コンバージョントラッキングは、a)広告ネットワーク上の広告主の広告のクリック、およびb)広告主のウェブサイトでの結果として生じるトランザクションを追跡するためにCookieを使用します。

**コストの精度:** (ポートフォリオを持つ広告主様)あるポートフォリオの実際の費用を予測費用で割ったもの。

**コストモデル:** (ポートフォリオを持つ広告主)Search、Social、および各ポジションまたは配置を獲得するために必要な入札、および履歴データおよび数学的予測手法を使用して各入札ユニットのCPC(検索)またはCPM(表示)を予測する、、およびコマーステクノロジー。

**コスト モデルのカバレッジ:** (ポートフォリオを持つ広告主様) 最適化機能がコスト モデルをビルドできるように、過去 7 日間に少なくとも 1 つのインプレッションを受信した CPC または eCPC キャンペーンの入札ユニットの数と割合。 すべての入札ユニットにコストモデルがあるわけではありません。コスト モデルを持つ入札単位は、コスト モデル カバレッジにカウントされます。

**コストモデルの半減期:** (ポートフォリオを持つ広告主様) 現在の日付より前の日数で、コストデータが最新であると見なされ、コストモデルとの関連性が高まります。

**1,000 インプレッションあたりのコスト:** (CPM) 1,000 インプレッションごとの 1 つの広告のコスト。 CPM価格設定モデルを使用している広告主様は、クリック数ではなくインプレッション数でお支払いをおこないます。

**獲得:** あたりのコスト(CPA) 広告のコストをコンバージョン数で割った値。 トランザクション単価(CPT)、注文単価(CPO)とも呼ばれます。

**クリック単価:** (CPC) 1) 広告のコストをその広告の合計クリック数で割った値。 たとえば、広告インプレッションに 100 米ドルを費やし、広告で 10 回のクリックが発生した場合、クリック単価は 100 USD/10=10 USD クリックごとになります。 2)広告主が広告クリックごとに課金される価格設定モデル。

**注文あたりのコスト:** (CPO) 広告のコストを注文数で割ったもの。 獲得あたりのコスト(CPA)またはトランザクションあたりのコスト(CPT)とも呼ばれます。

**トランザクション:** あたりのコスト(CPT) 広告のコストをトランザクション数で割ったもの。 獲得あたりのコスト(CPA)とも呼ばれます。

**CPA:** 「獲得あたりのコスト」を参照してください。

**CPC:** 「クリック単価」参照

**CPM:** 「1,000 インプレッションあたりのコスト」を参照してください。

**CPO:** 「注文あたりのコスト」を参照してください。

**CPT:** 「トランザクションあたりのコスト」を参照してください。

**CSV:** カンマ区切り値(CSV)で構成されるファイル形式。

**CTR:** 「クリックスルー率」を参照してください。

## E-F {#e-f}

**eCPM:** 指定した日付範囲の有効CPM、または 1,000 インプレッションあたりに支払われた平均コスト。 eCPM 値は、CPMキャンペーンまたはCPCキャンペーンに対して計算できます。

## I-J {#i-j}

**インプレッション:** Web ページ、モバイルアプリまたはその他の配信メディア上の広告の単一ディスプレイ。 ユーザー広告を表示したりクリックしたりしなくても、インプレッションとしてカウントされます。

**インプレッション ルックバックウィンドウ:** (表示およびソーシャルキャンペーンのみ) 広告インプレッション発生後にインプレッションをコンバージョンに結び付けることができる日数を指定する広告主レベルの設定。

**インプレッション オーバーライド 重み付け:** コンバージョンの前に有料クリックとインプレッションの両方が先行している場合に、クライアントのインプレッションルックバックウィンドウ内に発生したインプレッションに帰属するコンバージョン値の指定されたパーセンテージ。 コンバージョンの前にインプレッションのみがある場合、インプレッションオーバーライド重み付けではなく、広告主のビュースルーの重みがインプレッションに適用されます。

## K-L {#k-l}

**キーワード:** 広告に関連付けられた単語または語句。

**キーワード制約:** 「制約」を参照してください。

**ラベル 分類:** アカウントコンポーネントを意味のあるセットにグループする方法。 ラベル分類には、属性を表す複数のラベル値を含めることができます。 例えば、「地域」ラベル分類に異なる地理的地域の値を含めることができます。

**ラベル値:** ラベル分類の要素。 キャンペーンとキャンペーンエンティティ広告タグとして割り当てることができるため、パフォーマンスデータとレポートをラベルでフィルタリングしたり、ラベルに関連付けられた入札単位にオプションの制約を設定したりできます。

**ランディングページ URL:** 広告主の Web ページのURL。ユーザーの終わりに広告をクリックすると開きます。

## M-N {#m-n}

**限界費用:** 量が 1 単位変化した場合の総コストの変化。

**限界コスト対目標値:** 目標値を1つ上げるために必要なコストの変化。 この値は、従来の列「限界費用対収益」と同じです。

**一致タイプ:** 検索用語を広告と照合する方法を指定するオプション。 オプションは広告ネットワークによって異なります。

**最小 入札:** 1)インプレッションまたは 1,000 インプレッションあたりに支払う最低支払額。 2)検索キーワードの場合、品質スコアに基づいて特定のキーワードに最低限必要な入札。 通常、最低入札額は、キーワードに広告を表示するために支払うことができる最低クリックごと額です。

## O-P {#o-p}

**目的:** (ポートフォリオを持つ広告主)クライアントが特定のポートフォリオまたはディスプレイキャンペーンのビジネス目標を達成するために設定する目標(利益の最大化や特定の販売ターゲットの達成など)。 目標は、ポートフォリオに合わせて追跡および最適化されるコンバージョンメトリックと、それらのメトリックの相対的な重みで構成されます。

**目標値:** (ポートフォリオを持つ広告主様) ポートフォリオ現在の目標に従って計算された、次のものを含む、重み付けされたコンバージョンの合計。

* a)ポートフォリオの目的関数で各コンバージョンに割り当てられた重み、および該当する場合は表示表示スルーのビュースルーの重みアカウントを含むすべての変換。

* すべてのクリック数: 最適化機能では 1 つのコンバージョンと見なされ、目標のクリック値に基づいて重み付けされます。

この値は、従来の列「加重売上高」と同じ値になります。

**最適化機能:** (ポートフォリオを持つ広告主)Search、Social、コマースキーワード入札テクノロジーは、ビジネス目標に基づいてポートフォリオに最適な入札戦略と予算管理戦略を決定します。

**オーファントランザクション:** 特定のキーワードまたは広告に関連付けることができないトランザクション イベント。

**pixel:** トラッキング Web ページに埋め込まれた、1 x 1 ピクセルの透明な画像。 Adobe Systems Advertising の コンバージョン-トラッキング タグには、クリックとその結果のトランザクションをトラッキングするためのHTML画像ピクセルまたはJavaScriptが含まれています。

**配置:** ディスプレイ ネットワーク上で広告が表示される場所。 Web サイト全体、Web サイトのサブセット、または特定のページ上の広告位置を指定できます。

**ポートフォリオ:** 単一のビジネス目標とパフォーマンスターゲットに合わせて最適化された、一連の広告キャンペーンとそれに関連する入札単位。

**POS:** 支出の割合

**PPC:** 「有料クリックごと」を参照してください。

**プロパティ:** 「コンバージョン指標」参照

**プロパティ時間:** 個人 コンバージョン イベントが発生した時間。 イベントに関連するフォローするオン イベント (顧客が最初に無償試用版に登録し、後で有料サービスにサブスクライブするなど) が含まれている場合、各イベントには独自のプロパティ時間があります。

## Q-R {#q-r}

**品質スコア:** 広告ネットワークがキーワードの 1 つに割り当てられ、入札価格と広告掲載順位を決定するスコア。 これは、キーワードに関連する広告、ユーザー検索クエリー、キーワードのクリック率、およびその他の要因との関連性に基づいて計算されます。

**リダイレクト URL:** 広告主ランディングページの前、またはの代わりに、ユーザーを別のサーバーに送信する宛先URLの一部。

**投資回収率:** (ROI)売上高からコストを引いたもの。

**売上高精度:** (ポートフォリオを持つ広告主様)ポートフォリオの実際の売上高を予測売上高で割った値。

**売上高モデル:** (ポートフォリオを持つ広告主)クリックデータ(検索およびソーシャル)またはインプレッションデータ(表示)および広告主コンバージョンデータに基づいて、各入札ユニットのコンバージョン率と推定リターンを予測するSearch、Social、およびコマーステクノロジー。

**売上高モデルのカバレッジ:** (ポートフォリオを持つ広告主)売上高モデルを含むポートフォリオ内の入札ユニットの数および割合。 入札ユニットは、売上高を受け取っていないがインプレッションを獲得している場合は、売上高モデル均等を持つことができます。

**売上高 モデルの半減期:** (ポートフォリオを持つ広告主) 現在の日付より前の日数で、売上高データが新しいと見なされ、売上高モデルとの関連性が高くなります。

**ROI:** 「投資回収率」参照

## S-T {#s-t}

**シミュレーション:** (ポートフォリオを持つ広告主)Portfolio、履歴データを使用して、さまざまなレベルの支出と対応する1日の予算でポートフォリオが期待できるクリック数とコンバージョン数を推定するモデリングです。

**予算戦略:** (ポートフォリオを持つ広告主) ポートフォリオに対する キーワード/広告 入札を最適化するために選択した戦略。

**`s_kwcid`:「AMO ID」を参照してください**

**トラッキング URL:** 広告のクリックに関する情報を追跡するためにパラメーターが追加されたトラッキング テンプレートまたは宛先のURL。 ユーザーを広告主のランディングページにリダイレクトする前に、まずユーザーをトラッキングサーバーに送るリダイレクト URLが含まれる場合があります。

**トランザクション:** オンラインまたはオフライン発生する追跡可能なイベント。 複数の トランザクション イベントを同じトランザクションまたはコンバージョンの一部としてまとめて追跡できます。たとえば、情報のオンライン リクエストと電話による注文の結果は、購入のコンポーネントと見なされる場合があります。 「トランザクション」と「コンバージョン」という用語は、しばしば同じ意味で使用されます。 トランザクションはトランザクション ID で表され、1 つ以上のプロパティが関連付けられています。

**トランザクション ID:** トランザクションを識別する、広告主指定の ID。 トランザクションに複数のイベントが含まれている場合、それらはすべて同じトランザクション ID を持ちます。

**トランザクション プロパティ:** 「コンバージョン」参照

**トランザクション時間:** クリックまたはインプレッションがトランザクションに変換される時間。 トランザクションが複数のトランザクション イベントで構成されている場合 (顧客が最初に無償試用版に登録し、後で有料サービスにサブスクライブする場合など)、トランザクション時間はチェーンの最初のイベント (無償 試用版への登録) から発生します。

**TSV:** タブ区切り値(CSV)で構成されるファイル形式。

## U-V {#u-v}

**表示スルー:** (表示広告とソーシャル広告) ユーザーが広告をクリックしなくてもコンバージョンになる広告インプレッション(またはインプレッションの文字列)。

**ビュースルーの重み:** (表示およびソーシャルキャンペーンのみ)クリックベースのコンバージョンに関連付けられる重み付けに対する、表示スルーコンバージョンに結び付ける重み付けをパーセンテージで指定する広告主レベルの設定。

## Y-Z {#y-z}

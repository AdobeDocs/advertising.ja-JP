# 広告ネットワークとオブジェクトでクリックトラッキング URL を生成するタイミングと方法

次の表に、様々なキャンペーンコンポーネントのクリックトラッキング URL を生成する方法を示します。

| 広告ネットワーク | コンポーネント | クリックトラッキング URL の生成方法 |
| ---- | ---- | ---- |
| [!DNL Baidu]、[!DNL Google Ads]、[!DNL Microsoft Advertising]、[!DNL Yahoo! Japan Ads] および [!DNL Yandex] | <ul><li>テキスト広告</li><li>キーワード</li><li>（[!DNL Google Ads] コンテンツキャンペーン）プレースメント</li><li>（[!DNL Google Advertising] および [!DNL Advertising]） サイトリンク</li></ul> | アクティブなキャンペーンのトラッキング設定に「[!UICONTROL EF Redirect]」および「[!UICONTROL Auto Upload]」オプション（キャンペーンレベルで設定されるか、アカウント設定から継承）が含まれている場合、広告グループコンポーネントのトラッキング URL を生成する必要はありません。 Commerce a） （最終 URL を持つアカウント） トラッキングテンプレートの検索、ソーシャル、Commerceのトラッキングパラメーターで、最終 URL に同じパラメーターが付加されたもの、b） （宛先 URL を持つアカウント）検索、ソーシャル、Commerceのトラッキングコードおよび c が埋め込まれた新しい宛先 URL） （[!DNL Google Ads] および [!DNL Microsoft Advertising] アカウント） ランディングページのサフィックス （最終 URL サフィックス） パラメーター。<br><br> 「[!UICONTROL Auto Upload]」オプションが無効になっている場合は、次のいずれかの方法でコンポーネントのトラッキング URL を生成できます。<ul><li>（[!DNL Google Ads]、[!DNL Microsoft Advertising]、[!DNL Yahoo! Ads]、[!DNL Yandex]）フィードファイルから広告を投稿する場合は、「[!UICONTROL Generate Tracking URLs]」オプションを選択します。 オプションで、広告ネットワークに投稿する前に、バルクシートファイルのトラッキングテンプレートフィールドを検証できます。</li><li>コンポーネントを含むバルクシートファイルをダウンロード、アップロード、投稿する場合は、「[!UICONTROL Generate Tracking URLs]」オプションを選択します。 宛先 URL があるアカウントの場合、ファイルを広告ネットワークに POST する前に、ベース URL/最終 URL フィールドを検証するオプションがあります</li><li>[[!UICONTROL Tracking URLs] ツールを使用して &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) トラッキング URL を生成し、適切な [!UICONTROL Tracking Template] または [!UICONTROL Base URL] フィールドに手動で追加します。 <b> メモ：</b> 生成するトラッキングテンプレートには、アカウント設定またはキャンペーン設定で指定された追加のトラッキングパラメーターは含まれません。<ul><li>Google アカウント） [[!UICONTROL Tracking URLs] ツールに移動し &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) 画面上の値を適切な [!UICONTROL Tracking Template] フィールドにコピーして、トラッキング文字列全体を手動でコンポーネント設定に追加します。 最終的な URL の [!DNL Google Ads] [!DNL ValueTrack] パラメーターを `&url=` パラメーターの後に追加する必要があります（`{lpurl}` など）。 トラッキングテンプレートで最終的な URL を示す [!DNL ValueTrack] パラメーターのリストについては、[[!DNL Google Ads] ドキュメント ]9https://support.google.com/google-ads/answer/2375447の「使用可能な [!DNL ValueTrack] パラメーター」の節の「トラッキングテンプレートのみ」のパラメーターを参照してください。</li><li>（最終的な URL を持つその他のアカウント） [[!UICONTROL Tracking URLs] ツールを使用してトラッキング URL を生成し &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) トラッキング文字列全体をコンポーネント設定に手動で追加します。 最終的な URL のパラメーターを `&url=` パラメーターの後に追加する必要があります（`{lpurl}` など）。 [!DNL Yahoo! Japan Ads] アカウントの場合は、パラメーター `{lpurl}` を使用します。 トラッキングテンプレートで最終的な URL を示す [!DNL Microsoft Advertising] パラメーターのリストについては、[[!DNL Microsoft Advertising]  ドキュメント &#x200B;](https://help.bingads.microsoft.com/#apex/3/en/56799) を参照してください。</li><li>（宛先 URL を持つアカウント） [[!UICONTROL Tracking URLs] ツールを使用してトラッキング URL を生成し &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) 適切な [!UICONTROL Base URL] フィールドにトラッキング URL を手動で追加します。</li></ul></li></ul><b> 注：</b><br><br><ul><li>（最終的な URL を持つアカウント）最も詳細なレベルでの追跡テンプレートが使用されます（例えば、キーワードレベルの追跡テンプレートはアカウントレベル、キャンペーンレベルおよび広告グループレベルのテンプレートを上書きし、キーワードと配置の追跡テンプレートは関連する広告のテンプレートを上書きします）。</li><li>Adobe Advertisingは、サイトリンクのクリック数と結果の売上高を、個別にではなく、サイトリンクが含まれる広告に関連付けられたキーワードにマップします。 「[&#x200B; サポート対象インベントリ &#x200B;](/help/search-social-commerce/introduction/supported-inventory.md)」を参照してください。</li></ul><b> ヒント：</b> （最終的な URL を持つアカウント）トラッキングは、必要な最高レベルのトラッキングテンプレート（例えば、アカウントレベルまたはキャンペーンレベルのトラッキングテンプレート）のみを作成し、アカウントまたはキャンペーン内のすべてのエンティティに同じトラッキングを適用する場合、管理が最も簡単です。 |
| [!DNL Google Ads] | 電話の内線番号 | 該当なし |
| [!DNL Google Ads], [!DNL Microsoft Advertising] | 製品広告 | <ul><li>[!DNL Microsoft Merchant Center] アカウント：[&#x200B; ショッピング広告のトラッキングテンプレート形式 &#x200B;](/help/search-social-commerce/tracking/formats-click-tracking-microsoft.md) を使用して、[!DNL Microsoft Merchant Center] アカウントの各製品のトラッキング URL を手動で作成し、アカウント、キャンペーン、製品グループの設定の [!UICONTROL Tracking Template] フィールドに手動で追加します。<br><br> または、トラッキング URL を [!DNL Microsoft Merchant Center account] 内の製品データに追加できます。 これを行うには、トラッキング URL を、必要に応じて「[!DNL link]」または「[!DNL mobile_link]」フィールドの値と共に [&#x200B; 製品フィード内のカスタム列「[!DNL bingads_redirect]」に含め &#x200B;](https://help.ads.microsoft.com/#apex/3/en/51084) す。 「[!DNL bingads_redirect]」フィールドの値は、「[!DNL link]」および「[!DNL mobile_link]」フィールドの値を置き換えます。 この方法を使用して生成された URL には、アカウント設定で指定されたトラッキングパラメーターは含まれません。<br><br><b> メモ：</b> 同期中にトラッキングを自動的にアップロードするアカウントレベルおよびキャンペーンレベルの機能では、新しい [!DNL Microsoft Advertising] 製品グループのトラッキングは生成されません。 回避策は、バルクシートをアップロードまたは投稿する際にトラッキングを生成することです。</li><li>[!DNL Google Merchant Center] アカウント：[[!UICONTROL Tracking URLs] ツールを使用してトラッキング URL を生成し &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) アカウント、キャンペーンまたは製品グループ設定の [!UICONTROL Tracking Template] フィールドに手動で追加します。</li></ul> |
| [!DNL Naver] | キーワード | [&#x200B; バルクシート &#x200B;](/help/search-social-commerce/campaign-management/bulksheets/bulksheet-about.md) を使用して、すべての広告のクリック追跡を設定できます。 または、広告のトラッキング URL を手動で生成し、広告ネットワークのエディターを使用して手動で広告設定に追加することもできます。 [&#x200B; 実装  [!DNL Naver]  トラッキング専用アカウント &#x200B;](/help/search-social-commerce/campaign-management/naver-tracking-only-account-implement.md)」を参照してください。 |
| [!DNL Yahoo! Display Network] | テキスト広告とディスプレイ広告 | アクティブなキャンペーンのトラッキング設定に「[!UICONTROL EF Redirect]」および「[!UICONTROL Auto Upload]」オプション（キャンペーンレベルで設定されるか、アカウント設定から継承）が含まれている場合、広告のトラッキング URL を生成する必要はありません。 検索、ソーシャル、Commerceでは、トラッキングコードが同期されるたびに、新しいリンク先 URL が自動的に作成され、アドネットワークにアップロードされます。<br><br> 「[!UICONTROL Auto Upload]」オプションが無効になっている場合は、[[!UICONTROL Tracking URLs] ツールを使用してトラッキング URL を生成し &#x200B;](/help/search-social-commerce/tools/click-tracking-url-generate.md) 広告ネットワークのエディターを使用して、手動で広告設定に追加できます。 |

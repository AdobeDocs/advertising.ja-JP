# トラッキングに関する FAQ

## トラッキング機能

+++管理対象外のキャンペーンをAdobe Advertisingで追跡できますか？

はい。 検索、ソーシャル、コマースが、いずれかの広告ネットワークアカウントを同期している場合、すべての広告ネットワークのクリックデータを追跡します [サポートされるキャンペーンタイプ](/help/search-social-commerce/introduction/supported-inventory.md) というのも、その考え方です。 また、広告やキーワードのリンク先 URL やトラッキングテンプレートに検索、ソーシャル、コマースのリダイレクトを追加し、コンバージョンページにコンバージョントラッキングを実装している場合は、コンバージョンデータも追跡します。 検索、ソーシャル、コマースで追跡するキャンペーンと、どのキャンペーンをAdobeに管理してもらいたいかを、ユーザーアカウントチームで明確に示します。
+++

+++マルチイベントアトリビューションを取得する方法を教えてください。

検索、ソーシャル、コマース、Adobe Analyticsのコンバージョントラッキングタグを使用する広告主の場合、Adobe広告は、コンバージョンにつながる一連のイベントをまたいでコンバージョンデータを関連付ける複数のオプションを提供します。 広告主レベルの設定では、チャネルがイベントレベルでインプレッショントラッキングを許可している限り、複数の広告チャネルで発生した場合でも、イベントをまたいでコンバージョンデータをアトリビュートする方法を決定します。 デフォルトでは、コンバージョンは最後の（最新の）イベントに関連付けられますが、最初のイベントにコンバージョンを関連付けたり、すべてのイベントに均等に重みを付けたりするなど、設定は異なる方法で指定できます。 アトリビューションルールを変更すると、将来の入札の計算方法に影響します。

フィードファイル内のすべてのコンバージョンデータを提供する広告主は、コンバージョンを、関連するトランザクションイベント自体の属性にする必要があります。

>[!NOTE]
>
>キャンペーン管理ビュー、ポートフォリオ管理ビュー、レポート、（一部のユーザーが利用可能な）カスタムシミュレーションでは、将来の入札の計算方法に影響を与えることなく、ビューとレポート結果内のコンバージョンデータの属性付けに使用するルールを変更できます。

+++

+++Adobe Advertisingで重複トランザクションを識別する方法

トランザクションの完了後にユーザーが確認ページを更新すると、重複トランザクションが発生する場合があります。 Adobe広告は `ev_transid` 属性を使用して、同じトランザクション ID とプロパティ値を持つ重複トランザクションを排除します。

次に、Adobe広告の重複除外ロジックを示します。

* **クライアントが `ev_transid` 属性：** 以降のピクセルリクエストは、以下がすべて同じ場合、前のピクセルリクエストの重複と見なされます。の `ev_transid`;同じキーワード、広告またはプレースメントのトラッキング ID。特定のトランザクションプロパティの値。

  例えば、特定の広告ネットワーク上で、複数のローン申し込みの申し込み ID と同じキーワードのローン金額が同じ場合、それらは重複と見なされ、最初のローン申し込みのみがカウントされます。

* **クライアントが `ev_transid` 属性：** 後続のトランザクションは、同じキーワード、広告またはプレースメントのトラッキング ID を共有する場合、前のトランザクションの重複と見なされます。特定のトランザクションプロパティに対して同じ値を持つ。

  例えば、複数のローン申し込みのキーワード ID とローン金額が同じ場合、重複と見なされ、最初のローン申し込みのみがカウントされます。
+++

## トラッキング実装のタイプ

+++1 つ以上のキャンペーンまたはアカウントに対して、Adobe Advertisingコンバージョントラッキングサービスの使用を停止します。 トラッキング URL からトラッキングコードをすばやく削除するにはどうすればよいですか？

まず、トラッキング URL を削除することがもたらす影響については、Adobeのアカウントチームにお問い合わせください。

アカウントまたはキャンペーンで、トラッキング方法を「[!UICONTROL No EF Redirect].&quot; 次に、[!UICONTROL Generate Tracking URLs]」オプションを選択し、広告ネットワークに投稿します。 既存のトラッキング URL またはリンク先 URL はすべて置き換えられます。

+++孤立トランザクションとは

孤立トランザクションとは、特定のキーワードや広告に関連付けることができないトランザクションイベントです。 Adobe広告は、受け取ったトラッキング ID をキーワードのトラッキング URL または広告のトラッキング ID と照合することで、トランザクション/売上高をキーワードまたは広告に属性付けします。

Adobeのアカウントチームが孤立したトランザクションが売上の減少に起因すると疑うと、カスタマーケアチームは孤立をチェックし、見つかった場合は問題を調査します。

孤児は次のような状況で生じます。

**ピクセル実装**

孤立トランザクションは、ピクセル実装ではほとんど発生しません。 ただし、ピクセル孤立は次の場合に発生しています。

* クリックデータは、コンバージョンのクリック日には使用できません。 この場合、検索、ソーシャル、コマースが広告ネットワークからクリックデータを取得すると、データが再びマッピングされます。

* クリックログは、変換ログの前に処理されません。

**フィードの実装**

* フィードに送信されるトラッキング ID は、Search、Social および Commerce が把握していないアカウントからのものです。

* アカウントが同期されていないか、同期プロセス中にエラーが発生しました。

* Search、Social、および Commerce が広告ネットワークと同期していない場合に、トラッキング ID が広告ネットワークに追加および削除されたので、Search、Social、および Commerce は、この ID に関する情報を持ちません。 この問題は、売上高の受信に遅延がある場合、孤立した売上高を引き続き発生させます。

* クライアントは、フィードで正しくフォーマットされておらず、URL のトラッキング ID と一致しないトラッキング ID を送信しました。 これは、通常、フォーマットの問題や、フィードで追跡 ID が省略された場合に発生します。

* 設定ファイルで、URL からトラッキング ID を抽出するために使用される正規表現が正しくないか、古くなっています。 広告主が URL 内のトラッキング ID を変更したり、まったく新しいトラッキングシステムを採用したりする場合があります。その場合は、Search、Social、および Commerce の実装チームが正規表現を更新する必要があります。 このような場合、売上高の大部分は孤立しています。

**トランザクション ID を使用したフィード実装**

オフラインフィードでデータが使用可能な日付より前に使用できるオンライントランザクションはありません。

**トークン (ef_id) を使用したフィード実装**

Search, Social, &amp; Commerce は、サーバーまたは広告ネットワーク上で、対応するクリックを見つけることができません。 これは、クリックデータがコンバージョンのクリック日に使用できない場合や、（ほとんどの場合）クリックログがコンバージョンログより前に処理されなかった場合です。 検索、ソーシャル、コマースが広告ネットワークからクリックデータを受け取るか、クリックログが処理されると、データがコンバージョンにマッピングされます。
+++

+++複数のキーワードが同じトラッキング URL を持つ。

**影響**

複数の追跡 ID を持つフィード実装の場合、複数のキーワードに関するデータがレポートされます。 システムによって任意にいずれかのキーワードに変換が割り当てられるので、他の実装では、変換が誤ったキーワードに起因する場合があります。

**考えられる原因**

新しいキーワードまたは広告の URL が別のキーワードまたは広告からコピーされました。

これは、ディスプレイ広告やソーシャル広告では発生しません。

**考えられる解決策または回避策**

* 独自のキーワードおよび広告を管理する場合は、重複する URL に対する正しい URL を持つバルクシートファイルを作成し、 **[!UICONTROL Generate Tracking URLs]** 」オプションを使用します。すべてのキーワードと広告の URL が再生成されます。

* Adobeアカウントチームがキーワードを管理する場合は、重複する URL の新しい URL を作成するように依頼します。

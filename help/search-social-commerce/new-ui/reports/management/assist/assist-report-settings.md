# レポート設定の支援

*検索、ソーシャル、およびCommerceのクリック追跡と、Adobe AdvertisingとAdobe Analyticsのコンバージョン追跡を備えた広告主（[!DNL Analytics]統合）、またはトークン（`ef_id`）のみを使用したフィードで提供された広告主*

| Tab | パラメーター | 説明 |
|----|----|----|
| なし | [!UICONTROL Report Name] | （オプション）レポートとテンプレートの名前（レポートをテンプレートとして保存する場合）。 既存のテンプレートを適用する場合、テンプレート名はデフォルトで入力されます。 テンプレートを適用しない場合や名前を入力しない場合、レポートにはデフォルトで`<client name>-<date and time>-<report type>` （「acme - Apr 3, 2009 11:25:19 AM PDT - Keyword」など）という名前が付けられます。<br><br>任意でカスタム名を入力できますが、ファイル拡張子は使用しないでください。<br><br>FTP ディレクトリにレポートを送信するテンプレートを作成する場合は、ファイル名の任意の場所にオプションで「CSV」（大文字）を含めて、デフォルトのTSV形式ではなくCSV形式でファイルを作成できます。 FTP ディレクトリ [&#128279;](/help/search-social-commerce/new-ui/reports/ftp-reports.md)に送信されるレポートについては、 ファイル名の要件を参照してください。 |
|  | [!UICONTROL Save as Template] | （スケジュールに従ってレポートを実行しない限りオプション）レポート設定をテンプレートとして保存します。テンプレートは、[!UICONTROL Reports] > [!UICONTROL Report Templates] ビューで使用でき、新しいレポートの作成に再利用できます。 レポートをテンプレートとして保存するには、チェックボックスをオンにします。<br><br> スケジュールに従ってレポートを実行するには、設定をテンプレートとして保存する必要があります。<br><br><b> メモ：</b>既存のテンプレートに基づく場合でも、現在のパラメーターのセットを新しいテンプレートとして保存できます。 |
|  | [!UICONTROL Type] | 生成するレポートタイプ。 |

|  | [!UICONTROL Date Range] | データを生成する日付範囲：<ul><li><i>[!UICONTROL Custom Date Range]:</i>開始日と終了日を指定します。 MM/DD/YYYYまたはM/D/YYYY形式で日付を入力するか、フィールドの横にある![&#x200B; カレンダー](/help/search-social-commerce/assets/calendar-new.png " カレンダー")をクリックして日付を選択します。</li><li><i>[ プリセット範囲]:</i>一般的な時間増分のリスト（<i>[!UICONTROL Today]</i>から<i>[!UICONTROL Last 180 Days]</i>まで）。 デフォルトは<i>[!UICONTROL Last 7 Days]</i>で、データが利用可能な過去7日間をレポートします。 <b> メモ：</b> <i>[!UICONTROL Last Month]</i>、<i>[!UICONTROL Last 3 Months]</i>および<i>[!UICONTROL Last 6 Months]</i>は、過去1か月のデータを表示します。</li></ul> |
|  | [!UICONTROL Filter By] | （[!UICONTROL Campaign Assist Report]のみ）特定のポートフォリオまたは特定の広告ネットワークのデータについてレポートするかどうか：<ul><li><i>[!UICONTROL Portfolio]</i> （デフォルト）: キャンペーンのデータを1つ以上のポートフォリオに含めるには。</li><li><i>[!UICONTROL Search Engine]:</i> レポートの種類に応じて、キャンペーンのデータを1つ以上の広告ネットワークに含めるには、次の手順を実行します。 |

| [!UICONTROL Columns] | [!UICONTROL Path Size] | パターン内のアイテム（イベントタイプ、キーワードまたはプレースメント、広告グループ、キャンペーン）の最小数と最大数。 デフォルトのパスサイズは、1つ以上の項目と5つまでの項目です。 オプションで、2つ以上の項目を含むパスのみを表示できます。 アイテムのタイプと最大項目数は、レポートによって異なります。<ul><li>[!UICONTROL Channel Assist Report]：広告主の[&#x200B; クリックのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d)および[&#x200B; インプレッションのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j)内で発生したコンバージョンパスの最も早いイベントを最大N個までデータを列挙します。 例えば、1つ以上5つ以上のパスサイズを選択した場合、レポートには、最大5つのイベントを含むコンバージョンパスが含まれ、追跡されたイベントタイプのパターンごとに1行が含まれます（「検索クリック」や「表示画像」など）。 パスには最大30個のイベントを含めることができます。</li><li>[!UICONTROL Keyword Assist Report]：広告主の[&#x200B; クリックのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d)および[&#x200B; インプレッションのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j)内で発生したコンバージョンパス内のN個までの検索キーワードまたはプレースメントのデータを列挙します。 例えば、1つ以上5つ以上のパスサイズを選択した場合、レポートには、最大5つのキーワードまたはプレースメントを含むパスが含まれ、追跡されるキーワード文字列またはプレースメントのパターンごとに1行が含まれます。 パスには最大10個のイベントを含めることができます。</li><li>[!UICONTROL Campaign Assist Report]：広告主の[&#x200B; クリックのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#c-d)および[&#x200B; インプレッションのルックバックウィンドウ &#x200B;](/help/search-social-commerce/glossary.md#i-j)内で発生したコンバージョンパスの最も早いキャンペーンのデータを列挙します。 例えば、1つ以上5つ以上のパスサイズを選択した場合、レポートには、最大5つのキャンペーンを含むパスが含まれ、追跡されたキャンペーンのパターンごとに1行が表示されます。 パスには、最大10件のキャンペーンを含めることができます。</li></ul><b> メモ：</b><ul><li>アシストレポートのコンバージョンデータを表示するには、適切なコンバージョン列を追加する必要があります。</li><li>コンバージョンパスに、レポートで列挙しているイベントタイプ、キーワードまたはプレースメント、広告グループ、またはキャンペーンが含まれている場合、レポートには、項目の数が多いコンバージョンのデータを要約する追加の行（10を超えるイベントのコンバージョンの場合は1行など）が含まれます。</li><li>2つ以上の項目を含むパスを含める場合の合計は、1つ以上の項目を含むパスの合計よりも低くなる可能性があります。</li></ul> |
| [!UICONTROL Columns] | [!UICONTROL Report Columns] | レポートに表示されるデータの列とその順序：<ul><li>列を追加するには、左側の列の指標名をクリックし、![右向き矢印](/help/search-social-commerce/assets/arrow-right-customize.png "右向き矢印")をクリックします。</li><li>列を削除するには、右側の列の指標名をクリックし、![左向き矢印](/help/search-social-commerce/assets/arrow-left-customize.png "左向き矢印")をクリックします。</li><li>レポート内で列を左に移動するには、右側の列の指標名をクリックし、![上向き矢印](/help/search-social-commerce/assets/arrow-up-customize.png "上向き矢印")をクリックします。</li><li>レポート内で列を右に移動するには、右側の列の指標名をクリックし、![下向き矢印](/help/search-social-commerce/assets/arrow-down-customize.png "下向き矢印")をクリックします。</li></ul><p><b> メモ：</b></p><ul><li>アシストレポートのコンバージョンデータを表示するには、適切なコンバージョン列を追加する必要があります。</li><li>特定の種類のデータのみをリストするには、リストの上にあるアイコンのいずれかをクリックします。<ul><li>[!UICONTROL Campaign Status]などの広告ネットワークアカウントまたはポートフォリオコンポーネントのプロパティ名とIDの![&#x200B; プロパティ &#x200B;](/help/search-social-commerce/assets/property-columns.png " プロパティ ")</li><li>広告主が追跡するコンバージョン指標（Adobe Analyticsから同期されたコンバージョン指標やサイトエンゲージメント指標を含む）の![収益指標](/help/search-social-commerce/assets/revenue-metric-columns.png "収益指標")</li><li>広告主によって作成されたカスタム派生指標の![派生指標](/help/search-social-commerce/assets/derived-metric-columns.png "派生指標")</li></ul></li><li>多くのコンバージョン指標や、多くのコンバージョン指標を含むカスタム派生指標を含むレポートでは、生成に時間がかかります。</li><li>新しい指標を追加、作成、または編集するには、「[&#x200B; カスタム指標を作成](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-create.md)」、「[&#x200B; カスタム指標を編集](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-edit.md)」、「[&#x200B; カスタム指標を削除](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-delete.md)」を参照してください。</li><li>レポートタイプ別のすべての使用可能な列については、「[The[!UICONTROL Channel Assist Report]](channel-assist-report.md)」、「[The [!UICONTROL Campaign Assist Report]](campaign-assist-report.md)」、「[The [!UICONTROL Keyword Assist Report]](keyword-assist-report.md)」を参照してください。</li></ul> |
|  | [!UICONTROL Order Results/Limit Rows by] | レポートに含まれる列を最大2つ並べ替えます。 デフォルト値は、レポートタイプごとに異なります。 並べ替え順序をカスタマイズするには、レポート列を選択し、<i>[!UICONTROL Ascending]</i> （A ～ Zまたは1 ～ 100の結果を表示する場合）または<i>[!UICONTROL Descending]</i> （Z ～ Aまたは100 ～ 1の結果を表示する場合）を選択します。 並べ替える列を少なくとも1つ指定してください。 2つの列で並べ替えた場合、レポートは最初に指定した最初の列、次に指定した2番目の列で並べ替えられます。 |
|  | [!UICONTROL Share with others] | 同じ広告主のデータにアクセスできる他のユーザーが生成されたレポートを表示し、レポートをテンプレートとして保存する場合は、テンプレートを使用しますが、編集や削除はしません。 デフォルトでは、このオプションは選択されていません。 <b>注：</b>この設定にかかわらず、レポートとテンプレートは、より上位（管理者）の役割を持つすべてのユーザーと、割り当てられたAdobe Account Team メンバーに対して常に表示されます。 |

|  | [!UICONTROL Indicate account name after entity name] | （[!UICONTROL Campaign Assist  Report]のみ）広告ネットワークアカウント名をキャンペーン名の後ろに角括弧で囲みます。 例：`<campaign name> [Google Adwords] [Account1]` |

| [!UICONTROL Filters] | [!UICONTROL Report Filters] | （[!UICONTROL Campaign Assist Report]のみ）指標の値が指定された条件を満たす場合にのみ、行を返します。 指標をレポートの列として含める必要はありません。 使用可能な指標のリストはレポートタイプによって異なりますが、広告主のカスタム派生指標、各検索エンジンおよびポートフォリオコンポーネントのIDとプロパティ名（[!UICONTROL Campaign ID]や[!UICONTROL Campaign Status]など）、広告主のコンバージョン指標、広告ネットワークのクリック関連の指標が含まれる場合があります。 使用可能な演算子には、<i>[!UICONTROL contains]</i>、<i>[!UICONTROL starts with]</i>、<i>[!UICONTROL equals]</i>、<i>[!UICONTROL is greater than]</i>、<i>[!UICONTROL is greater than or equal to]</i>、<i>[!UICONTROL is less than]</i>、<i>[!UICONTROL is less than or equal to]</i>、または<i>[!UICONTROL isn't equal to]</i>が含まれます。<br><br>1つ以上のフィルターを適用するには、次の操作を行います。<ul><li>指標と演算子を選択し、該当する値を入力します。 例えば、クリック数が100回を超えるキーワードのみを返すには、[!UICONTROL Clicks]を選択し、[!UICONTROL >]を選択して、入力フィールドに100を入力します。</li><li>（追加のフィルターを適用するには）追加の各フィルターについて、**[!UICONTROL +Add Filter]**&#x200B;をクリックし、**[!UICONTROL AND]**&#x200B;または&#x200B;**[!UICONTROL OR]**&#x200B;を選択し、指標と演算子を選択してから、該当する値を入力します。</li></ul> |

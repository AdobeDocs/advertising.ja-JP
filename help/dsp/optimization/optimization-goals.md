# 最適化目標とその使用方法

| 最適化目標 | 説明 | この目標を使用するタイミング |
| -----------| ---------- | ---------- |
| [!UICONTROL Always Max Bid & Highest Clickthrough Rate] | パッケージレベルの最適化を使用して、予算割り当ては、クリックスルー率が最も高いプレースメントを優先します。<br><br> オークション評価は、支出目標が満たされている CTR を優先します。 送信された入札は常に設定された [!UICONTROL Max Bid] ですが、プレースメントが適切に支出されている場合、予測される CTR しきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高のクリックスルー率 <br><br> 広告タイプ：プリロール、ディスプレイ <br><br><b> メモ：</b> この目標は、超過する必要のない固定 CPM 目標があり、最大化する必要がある CTR 目標がある場合に使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算を全額使い切ろうとして可能な限り最高の CTR を達成します。 |
| [!UICONTROL Always Max Bid & Highest Completion Rate] | パッケージレベルの最適化を使用して、予算割り当ては、完了率が最も高いプレースメントを優先します。<br><br> オークション評価の優先順位完了率支出目標が満たされている。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測される完了率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高の完了率 <br><br> 広告タイプ：プレロールのみ <br><br><b> 注意：</b> この目標は、超過する必要のない固定 CPM 目標があり、最大化する必要がある完了率目標がある場合に使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算をすべて使い切ろうとして、可能な限り最高の完了率を達成します。 |
| [!UICONTROL Always Max Bid & Highest Engagement Rate] | パッケージレベルの最適化を使用して、予算割り当ては、エンゲージメント率が最も高いプレースメントを優先します。<br><br> オークション評価では、支出目標が満たされているエンゲージメント率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるエンゲージメントレートのしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高のエンゲージメント率 <br><br> 広告タイプ：モバイルインタースティシャルのみ <br><br><b> メモ：</b> この目標は、超過する必要のない固定 CPM 目標があり、エンゲージメント目標を最大化する必要がある場合に使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算をすべて使い切ろうとして、可能な限り最高のエンゲージメントレートを達成します。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (Adobe - GroupM)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br> オークション評価では、支出目標が満たされているビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高のビューアビリティ率 <br><br> 広告タイプ：インタラクティブなプリロールのみ <br><br><b> 注意：</b> この目標では常にプレースメントレベルの最大入札額を使用します。<br><br> キャンペーンの「ビューアビリティ感度」設定が「標準（広告の 50% が 2 秒連続で表示される）」に設定されている場合、メディア評価評議会（MRC）のビューアビリティ測定標準がキャンペーンに使用されます。 キャンペーンを「厳密（広告の 100% ビュー内広告とオーディオの 50% 期間）」に設定した場合、キャンペーンには GroupM ビューアビリティ測定標準が使用されます。 キャンペーンの設定を、最適化目標および入札前フィルターの設定と一致させるのが理想です。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (Adobe - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br> オークション評価では、支出目標が満たされているビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高のビューアビリティ率 <br><br> 広告タイプ：インタラクティブなプリロールのみ <br><br><b> 注意：</b> この目標では常にプレースメントレベルの最大入札額を使用します。<br><br> キャンペーンの「ビューアビリティ感度」設定が「標準（広告の 50% が 2 秒連続で表示される）」に設定されている場合、メディア評価評議会（MRC）のビューアビリティ測定標準がキャンペーンに使用されます。 キャンペーンを「厳密（広告の 100% ビュー内広告とオーディオの 50% 期間）」に設定した場合、キャンペーンには GroupM ビューアビリティ測定標準が使用されます。 キャンペーンの設定を、最適化目標および入札前フィルターの設定と一致させるのが理想です。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (IAS - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br> オークション評価では、支出目標が満たされているビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：最高のビューアビリティ率 <br><br> 広告タイプ：インタラクティブなプリロールのみ <br><br><b> 注意：</b> この目標では常にプレースメントレベルの最大入札額を使用します。<br><br> この設定は、IAS のサードパーティデータがアルゴリズムに通知している場合に最適です。 この目標は、キャンペーンに対して IAS 統合を有効にした場合にのみ使用します。 |
| [!UICONTROL Always Max Bid and Maximize Reach] | この目標は、常にプレースメントレベルの [!UICONTROL Max Bid] を使用して、指定された数のインプレッションで最大の世帯リーチを達成することを試みます。 費用の目標が満たされている場合、DSPはより選択的になり、ユニークなリーチを段階的に達成する機会がある場合にのみオークションに入札します。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：リーチ <br><br> 広告のタイプを最大化：プリロール、ディスプレイ、CTV、ネイティブ、オーディオおよびユニバーサルビデオ |
| [!UICONTROL Highest Return on Ad Spend (ROAS)] | （パッケージレベルでのみ使用可能）予算割り当ては、カスタム目標に追加された重み付けされた上位ファネルイベント（web サイト訪問や買い物かごへの追加など）を考慮して、指定されたカスタム目標に含まれる最終コンバージョンイベントの ROAS が最も高いプレースメントに優先順位を付けます。 最適化モデルをクリックベースのコンバージョンからのみ学習するか、クリックベースのコンバージョンとインプレッションベースのコンバージョンの両方から学習するかを指定できます。<br><br> オークション評価では ROAS が優先されます。 費用の目標が達成される一方で、DSPは CPM の削減と ROAS の向上のバランスを取ります。 | キャンペーンタイプ：パフォーマンス <br><br> ベンチマーク：最も高い売上高 <br><br> 広告タイプ：ディスプレイ、ネイティブ、ビデオ、CTV およびユニバーサルビデオ <br><br><b> 注意：</b> 詳しくは、「[ パフォーマンスキャンペーンを設定するためのベストプラクティス ](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Acquisition (CPA)] | （パッケージレベルでのみ使用可能）予算割り当ては、カスタム目標に追加された重み付けされた上位ファネルイベント（web サイト訪問や買い物かごへの追加など）を考慮して、指定されたカスタム目標に含まれる最終コンバージョンイベントに対して、最も低い CPA のプレースメントを優先します。 最適化モデルをクリックベースのコンバージョンからのみ学習するか、クリックベースのコンバージョンとインプレッションベースのコンバージョンの両方から学習するかを指定できます。<br><br> オークション評価の優先順位付け CPA 費用の目標が満たされると、DSPは CPM の低下と CPA の低下のバランスを取ります。 | キャンペーンタイプ：パフォーマンス <br><br> ベンチマーク：最低の CPA<br><br> 広告タイプ：ディスプレイ、ネイティブ、ビデオ、CTV、ユニバーサルビデオ <br><br><b> 注意：</b> 詳しくは、「[ パフォーマンスキャンペーンの設定のベストプラクティス ](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Click] | パッケージレベルの最適化を使用して、予算割り当てでは CPC が最も少ないプレースメントが優先されます。<br><br> オークション評価は CPC を優先します。 費用の目標が満たされると、DSPは CPC を下げようとして CPM の低下と CTR の増加のバランスを取ります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高のクリックスルー率 <br><br> 広告タイプ：プリロール、ディスプレイ <br><br><b> 注意：</b> この目標を使用して、可能な限り最高の CPC を達成します。 最大 CPM を保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest Cost per Completion] | パッケージレベルの最適化を使用して、予算割り当ては、完了あたりのコストが最も低いプレースメントを優先します。<br><br> オークション評価では、ビデオ完了率（VCR）が優先されます。 費用の目標が満たされている場合、DSPは完了時のコストを削減するために、CPM の削減と VCR の増加のバランスを取ります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高の完了率 <br><br> 広告タイプ：プレロールのみ |
| [!UICONTROL Lowest Cost per Engagement] | パッケージレベルの最適化を使用して、予算割り当てでは、エンゲージメント率が最も低いプレースメントが優先されます。<br><br> オークション評価では契約率が優先されます。 費用の目標が満たされている場合、DSPは CPM の削減とエンゲージメントあたりのコストの削減のバランスをとろうとします。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高のエンゲージメント率 <br><br> 広告タイプ：モバイルインタースティシャルのみ |
| [!UICONTROL Lowest Cost per Reach] | この目標は、特定の予算で世帯の最大範囲を達成することを試みます。 支出目標が満たされている場合、ユニークなリーチを段階的に上げる機会によって入札は異なります。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：リーチ <br><br> 広告タイプあたりの効率的なコスト（プリロール、ディスプレイ、CTV、ネイティブ、オーディオ、ユニバーサルビデオ） |
| [!UICONTROL Lowest Cost per View] | 最小 CPM と同様に動作します。 パッケージレベルの最適化を使用して、予算割り当ては CPM が最も低いプレースメントを優先します。<br><br> オークション評価では CPM が優先されます。 費用の目標が満たされると、DSPは徐々に CPM を下げます。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高のクリックスルー率 <br><br> 広告タイプ：プレロール、ディスプレイ |
| [!UICONTROL Lowest CPM] | パッケージレベルの最適化を使用して、予算割り当ては CPM が最も低いプレースメントを優先します。<br><br> オークション評価では CPM が優先されます。 費用の目標が満たされると、DSPは徐々に CPM を下げます。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM<br><br> 広告タイプ：プリロール、ディスプレイ、CTV、ネイティブ、オーディオ |
| [!UICONTROL Lowest vCPM (Adobe - GroupM)] | パッケージレベルの最適化を使用して、予算割り当ては vCPM が最も低いプレースメントを優先します。<br><br> オークション評価では vCPM が優先されます。 費用の目標が満たされている場合、DSPは CPM の低下とビューアビリティの向上のバランスをとろうとします。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高の vCPM<br><br> 広告タイプ：プリロール、ディスプレイ <br><br><b> 注意：</b> この目標を使用して、可能な限り最高の vCPM を達成します。<br><br> 最大 CPM を保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest vCPM (Adobe - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては vCPM が最も低いプレースメントを優先します。<br><br> オークション評価では vCPM が優先されます。 費用の目標が満たされている場合、DSPは CPM の低下とビューアビリティの向上のバランスをとろうとします。 | キャンペーンタイプ：ブランディング <br><br> ベンチマーク：効率的な CPM と最高の vCPM<br><br> 広告タイプ：プリロール、ディスプレイ <br><br><b> 注意：</b> この目標を使用して、可能な限り最高の vCPM を達成します。<br><br> 最大 CPM を保証するには、プレースメントの最大入札額として使用します。 |

# 最適化目標とその活用方法

| 最適化の目標 | 説明 | この目標を使用するタイミング |
| -----------| ---------- | ---------- |
| [!UICONTROL Always Max Bid & Clickthrough Rate] | パッケージレベルの最適化により、予算配分では、クリックスルー率が最も高い配置に優先順位を付けます。<br><br> オークション評価では、CTRの優先順位付けと支出目標の達成が優先されます。 送信された入札は常に設定された[!UICONTROL Max Bid]ですが、プレースメントが適切に使用されている場合、予測されるCTRしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：クリック率が最も高い<br><br>広告の種類：プレロール、ディスプレイ <br><br><b>注：</b>この目標は、CPMの目標を上回る必要がなく、CTRの目標を最大化する必要がある場合に使用します。 CPMの目標に最大入札額を設定すると、DSPは予算を最大限に活用しながら、可能な限り最高のCTRを達成します。 |
| [!UICONTROL Always Max Bid & Completion Rate] | パッケージレベルの最適化では、予算配分で、最も完了率の高い配置に優先順位をつけます。<br><br> オークション評価では、支出目標が達成された完了率を優先します。 送信された入札は常に設定された最大入札ですが、プレースメントがうまく使用されている場合、予測された完了率のしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：最も高い完了率<br><br>広告の種類：プレロールのみ<br><br><b>注：</b>この目標は、CPMの目標を上回る必要がなく、完了率の目標を最大化する必要がある場合に使用します。 CPMの目標に最大入札額を設定すると、DSPは予算を最大限に活用しながら、可能な限り最高の完了率を達成します。 |
| [!UICONTROL Always Max Bid & Engagement Rate] | パッケージレベルの最適化により、予算配分では、エンゲージメント率の高いプレースメントに優先順位を付けます。<br><br> オークション評価では、支出目標が達成されているエンゲージメント率を優先します。 送信された入札は常に設定された最大入札ですが、プレースメントがうまく使用されている場合、予測エンゲージメント率のしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：エンゲージメント率が最も高い<br><br>広告の種類：モバイル間質性のみ<br><br><b>注：</b>この目標は、CPMの目標を上回る必要がなく、エンゲージメントの目標を最大化する必要がある場合に使用します。 CPMの目標に最大入札額を設定すると、DSPは予算を最大限に活用しながら、可能な限り最高のエンゲージメント率を達成します。 |
| [!UICONTROL Always Max Bid & Maximize Reach] | この目標は、常に配置レベル [!UICONTROL Max Bid]を使用して、特定の数のインプレッションで最大限の世帯リーチを達成しようとします。 支出目標が達成された場合、DSPはより選択的になり、リーチを拡大できるチャンスがある場合にのみオークションに入札します。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：リーチを最大化<br><br>広告の種類：プレロール、ディスプレイ、CTV、ネイティブ、オーディオ、ユニバーサルビデオ |
| [!UICONTROL Always Max Bid & Viewability Rate (Adobe - GroupM)] | パッケージレベルの最適化では、予算配分で、最も表示率の高い配置を優先します。<br><br> オークション評価では、支出目標が達成されている視聴率を優先します。 送信された入札は常に設定された最大入札ですが、プレースメントが適切に使用されている場合、予測された表示率のしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：最高の閲覧率<br><br>広告の種類：インタラクティブ プレロールのみ<br><br><b>注：</b>この目標では、常にプレースメント レベルの最大入札額が使用されます。<br><br> キャンペーンの表示可能性感度の設定が「標準（広告の50%が2秒連続で表示）」に設定されている場合、キャンペーンにはMedia Rating Council （MRC）の表示可能性測定標準が使用されます。 キャンペーンを「厳格（100%の広告を表示し、オーディオをオンにして50%のデュレーションを再生）」に設定した場合、GroupMのビューアビリティ測定標準がキャンペーンに使用されます。 理想的には、キャンペーンの設定を最適化目標と入札前のフィルター設定と一致させる必要があります。 |
| [!UICONTROL Always Max Bid & Viewability Rate (Adobe - MRC)] | パッケージレベルの最適化では、予算配分で、最も表示率の高い配置を優先します。<br><br> オークション評価では、支出目標が達成されている視聴率を優先します。 送信された入札は常に設定された最大入札ですが、プレースメントが適切に使用されている場合、予測された表示率のしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：最高の閲覧率<br><br>広告の種類：インタラクティブ プレロールのみ<br><br><b>注：</b>この目標では、常にプレースメント レベルの最大入札額が使用されます。<br><br> キャンペーンの表示可能性感度の設定が「標準（広告の50%が2秒連続で表示）」に設定されている場合、キャンペーンにはMedia Rating Council （MRC）の表示可能性測定標準が使用されます。 キャンペーンを「厳格（100%の広告を表示し、オーディオをオンにして50%のデュレーションを再生）」に設定した場合、GroupMのビューアビリティ測定標準がキャンペーンに使用されます。 理想的には、キャンペーンの設定を最適化目標と入札前のフィルター設定と一致させる必要があります。 |
| [!UICONTROL Always Max Bid & Viewability Rate (IAS - MRC)] | パッケージレベルの最適化では、予算配分で、最も表示率の高い配置を優先します。<br><br> オークション評価では、支出目標が達成されている視聴率を優先します。 送信された入札は常に設定された最大入札ですが、プレースメントが適切に使用されている場合、予測された表示率のしきい値は厳しくなります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：最高の閲覧率<br><br>広告の種類：インタラクティブ プレロールのみ<br><br><b>注：</b>この目標では、常にプレースメント レベルの最大入札額が使用されます。<br><br>この設定は、IASのサードパーティデータがアルゴリズムに情報を提供している場合に最適に機能します。 この目標は、キャンペーンのIAS統合を有効にした場合にのみ使用します。 |
| [!UICONTROL Highest Return on Ad Spend (ROAS)] | （パッケージレベルでのみ利用可能）予算配分では、指定されたカスタム目標に含まれる最終的なコンバージョンイベントに対して、最も高いROASを持つ配置を優先します。この場合、カスタム目標に重み付けされた上funnelのイベント（web サイトへの訪問やショッピングカートの追加など）をすべて考慮します。 最適化モデルがクリックベースのコンバージョンからのみ学習するか、クリックベースとインプレッションベースの両方のコンバージョンから学習するかを指定できます。<br><br> オークション評価でROASが優先されます。 支出目標を達成すると、DSPはCPMの削減とROASの向上のバランスをとります。 | キャンペーンの種類：パフォーマンス <br><br> ベンチマーク：最高収益<br><br>広告の種類：ディスプレイ、ネイティブ、ビデオ、CTV、ユニバーサルビデオ <br><br><b>注：</b>詳細については、「[&#x200B; パフォーマンスキャンペーンの設定に関するベストプラクティス &#x200B;](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Acquisition (CPA)] | （パッケージレベルでのみ利用可能）予算配分は、指定されたカスタム目標に含まれる最終的なコンバージョンイベントに対して、最小CPAを持つ配置を優先し、カスタム目標に重み付けされた上funnelのイベント（web サイトへの訪問やショッピングカートの追加など）をすべて考慮します。 最適化モデルがクリックベースのコンバージョンからのみ学習するか、クリックベースとインプレッションベースの両方のコンバージョンから学習するかを指定できます。<br><br> オークション評価でCPAが優先されます。 cpaを下げることで支出目標を達成し、DSPはCPMを下げることとCPAを下げることをバランスさせます。 | キャンペーンの種類：パフォーマンス <br><br> ベンチマーク：最小CPA<br><br>広告の種類：ディスプレイ、ネイティブ、ビデオ、CTV、ユニバーサルビデオ <br><br><b>注：</b>詳しくは、「[&#x200B; パフォーマンスキャンペーンの設定に関するベストプラクティス &#x200B;](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Click] | パッケージレベルの最適化では、予算配分によって、最小のCPCでの配置が優先されます。<br><br> オークション評価でCPCが優先されます。 支出目標は達成された後、DSPは、CPCを下げるために、CPMを下げることと、CTRを上げることをバランスさせます。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高のクリック率<br><br>広告の種類：プレロール、ディスプレイ <br><br><b>注：</b>この目標を使用して、可能な限り最高のCPCを実現します。 最大CPMを保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest Cost per Completion] | パッケージレベルの最適化では、予算配分によって、完了単価が最も低いプレースメントに優先順位が付けられます。<br><br> オークション評価では、ビデオ完了率（VCR）が優先されます。 支出目標は達成されていますが、DSPでは、CPMの削減とVCRの増加をバランスさせ、完了当たりのコストを削減しようとします。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高の完了率<br><br>広告の種類：プレロールのみ |
| [!UICONTROL Lowest Cost per Engagement] | パッケージレベルの最適化では、予算配分によって、エンゲージメント率が最も低いプレースメントに優先順位を付けます。<br><br> オークション評価でエンゲージメント率が優先されます。 支出目標を達成したら、DSPを利用して、CPMの削減と、エンゲージメントあたりのコストの削減のバランスをとります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最も高いエンゲージメント率<br><br>広告の種類：モバイルインタースティシャルのみ |
| [!UICONTROL Lowest Cost per Reach] | この目標は、与えられた予算で最大限の世帯リーチを達成しようとします。 支出目標が達成されている場合、入札額はユニークリーチを拡大する機会によって異なります。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：リーチ単価<br><br>広告の種類：プレロール、ディスプレイ、CTV、ネイティブ、オーディオ、ユニバーサルビデオ |
| [!UICONTROL Lowest Cost per View] | Lowest CPMと同じように動作します。 パッケージレベルの最適化では、予算配分によって、最も低いCPMのプレースメントが優先されます。<br><br> オークション評価でCPMが優先されます。 目標を達成すると、DSPによってCPMが段階的に削減されます。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高のクリック率<br><br>広告の種類：プリロール、ディスプレイ |
| [!UICONTROL Lowest CPM] | パッケージレベルの最適化では、予算配分によって、最も低いCPMのプレースメントが優先されます。<br><br> オークション評価でCPMが優先されます。 目標を達成すると、DSPによってCPMが段階的に削減されます。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPM<br><br>広告の種類：プリロール、ディスプレイ、CTV、ネイティブ、オーディオ |
| [!UICONTROL Lowest vCPM (Adobe - GroupM)] | パッケージレベルの最適化では、予算配分によって、vCPMが最も低いプレースメントが優先されます。<br><br> オークション評価でvCPMが優先されます。 支出目標は達成された後、DSPはCPMの削減と可視性の向上のバランスを取ろうとします。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高のvCPM<br><br>広告の種類：プレロール、ディスプレイ <br><br><b>注：</b>この目標を使用して、可能な限り最高のvCPMを実現します。<br><br>最大CPMを保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest vCPM (Adobe - MRC)] | パッケージレベルの最適化では、予算配分によって、vCPMが最も低いプレースメントが優先されます。<br><br> オークション評価でvCPMが優先されます。 支出目標は達成された後、DSPはCPMの削減と可視性の向上のバランスを取ろうとします。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高のvCPM<br><br>広告の種類：プレロール、ディスプレイ <br><br><b>注：</b>この目標を使用して、可能な限り最高のvCPMを実現します。<br><br>最大CPMを保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest vCPM (IAS - MRC)] | パッケージレベルの最適化では、予算配分によって、vCPMが最も低いプレースメントが優先されます。<br><br> オークション評価でvCPMが優先されます。 支出目標は達成された後、DSPはCPMの削減と可視性の向上のバランスを取ろうとします。 | キャンペーンの種類：ブランディング <br><br> ベンチマーク：効率的なCPMと最高のvCPM<br><br>広告の種類：プレロール、ディスプレイ <br><br><b>注：</b>この目標を使用して、可能な限り最高のvCPMを実現します。<br><br>最大CPMを保証するには、プレースメントの最大入札額として使用します。<br><br>この設定は、IASのサードパーティデータがアルゴリズムに情報を提供している場合に最適に機能します。 この目標は、キャンペーンのIAS統合を有効にした場合にのみ使用します。 |

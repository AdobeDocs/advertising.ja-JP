# 最適化目標とその使用方法

| 最適化目標 | 説明 | この目標を使用するタイミング |
| -----------| ---------- | ---------- |
| [!UICONTROL Always Max Bid & Highest Clickthrough Rate] | パッケージレベルの最適化を使用して、予算割り当ては、クリックスルー率が最も高いプレースメントを優先します。<br><br>支出目標が満たされている場合、オークション評価では CTR が優先されます。 送信された入札は常にセットになります [!UICONTROL Max Bid]ただし、プレースメントが十分に支出されている場合、予測される CTR しきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高のクリックスルー率<br><br>広告タイプ：プリロール、ディスプレイ<br><br><b>注意：</b> この目標は、超過する必要のない固定 CPM 目標と、最大化する必要がある CTR 目標がある場合に使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算を全額使い切ろうとして可能な限り最高の CTR を達成します。 |
| [!UICONTROL Always Max Bid & Highest Completion Rate] | パッケージレベルの最適化を使用して、予算割り当ては、完了率が最も高いプレースメントを優先します。<br><br>支出目標が満たされている場合、オークション評価では完了率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測される完了率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高の完了率<br><br>広告タイプ：プリロールのみ<br><br><b>注意：</b> この目標は、超過する必要のない固定 CPM 目標と、最大化する必要がある完了率目標がある場合に使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算をすべて使い切ろうとして、可能な限り最高の完了率を達成します。 |
| [!UICONTROL Always Max Bid & Highest Engagement Rate] | パッケージレベルの最適化を使用して、予算割り当ては、エンゲージメント率が最も高いプレースメントを優先します。<br><br>費用の目標が満たされている場合、オークション評価では契約率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるエンゲージメントレートのしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高のエンゲージメント率<br><br>広告タイプ：Mobile Interstitial のみ<br><br><b>注意：</b> 超過する必要のない固定 CPM 目標と、最大化する必要のあるエンゲージメント目標がある場合は、この目標を使用します。 最大入札額を目的の CPM 目標に設定すると、DSPは、予算をすべて使い切ろうとして、可能な限り最高のエンゲージメントレートを達成します。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (Adobe - GroupM)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br>費用の目標が満たされている場合、オークション評価ではビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高のビューアビリティ率<br><br>広告タイプ：インタラクティブなプリロールのみ<br><br><b>注意：</b> この目標では、プレースメントレベルの最大入札額を常に使用します。<br><br>キャンペーンの「ビューアビリティ感度」設定が「標準（広告の 50% が 2 秒連続で表示される）」に設定されている場合、メディア評価評議会（MRC）のビューアビリティ測定標準がキャンペーンに使用されます。 キャンペーンを「厳密（広告の 100% ビュー内広告とオーディオの 50% 期間）」に設定した場合、キャンペーンには GroupM ビューアビリティ測定標準が使用されます。 キャンペーンの設定を、最適化目標および入札前フィルターの設定と一致させるのが理想です。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (Adobe - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br>費用の目標が満たされている場合、オークション評価ではビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高のビューアビリティ率<br><br>広告タイプ：インタラクティブなプリロールのみ<br><br><b>注意：</b> この目標では、プレースメントレベルの最大入札額を常に使用します。<br><br>キャンペーンの「ビューアビリティ感度」設定が「標準（広告の 50% が 2 秒連続で表示される）」に設定されている場合、メディア評価評議会（MRC）のビューアビリティ測定標準がキャンペーンに使用されます。 キャンペーンを「厳密（広告の 100% ビュー内広告とオーディオの 50% 期間）」に設定した場合、キャンペーンには GroupM ビューアビリティ測定標準が使用されます。 キャンペーンの設定を、最適化目標および入札前フィルターの設定と一致させるのが理想です。 |
| [!UICONTROL Always Max Bid & Highest Viewability Rate (IAS - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては、ビューアビリティ率が最も高いプレースメントを優先します。<br><br>費用の目標が満たされている場合、オークション評価ではビューアビリティ率が優先されます。 送信される入札は常に設定された最大入札額ですが、プレースメントが適切に支出されている場合、予測されるビューアビリティ率のしきい値は厳しくなります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：最高のビューアビリティ率<br><br>広告タイプ：インタラクティブなプリロールのみ<br><br><b>注意：</b> この目標では、プレースメントレベルの最大入札額を常に使用します。<br><br>この設定は、IAS のサードパーティデータがアルゴリズムに通知している場合に最適です。 この目標は、キャンペーンに対して IAS 統合を有効にした場合にのみ使用します。 |
| [!UICONTROL Always Max Bid and Maximize Reach] | この目標は、常にプレースメントレベルを使用して、指定された数のインプレッションで最大の世帯リーチを達成することを試みます [!UICONTROL Max Bid]. 費用の目標が満たされている場合、DSPはより選択的になり、ユニークなリーチを段階的に達成する機会がある場合にのみオークションに入札します。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：リーチの最大化<br><br>広告タイプ：プリロール、ディスプレイ、CTV、ネイティブ、オーディオおよびユニバーサルビデオ |
| [!UICONTROL Highest Return on Ad Spend (ROAS)] | （パッケージレベルでのみ使用可能）予算割り当ては、カスタム目標に追加された重み付けされた上位ファネルイベント（web サイト訪問や買い物かごへの追加など）を考慮して、指定されたカスタム目標に含まれる最終コンバージョンイベントの ROAS が最も高いプレースメントに優先順位を付けます。 最適化モデルをクリックベースのコンバージョンからのみ学習するか、クリックベースのコンバージョンとインプレッションベースのコンバージョンの両方から学習するかを指定できます。<br><br>オークション評価では、ROAS が優先されます。 費用の目標が満たされている場合、DSPは CPM の削減と ROAS の増加のバランスを取ります。 | キャンペーンタイプ : パフォーマンス<br><br>ベンチマーク：最高売上高<br><br>広告タイプ：ディスプレイ、ネイティブ、ビデオ、CTV およびユニバーサルビデオ<br><br><b>注意：</b> 参照先」[パフォーマンスキャンペーンの設定のベストプラクティス](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Acquisition (CPA)] | （パッケージレベルでのみ使用可能）予算割り当ては、カスタム目標に追加された重み付けされた上位ファネルイベント（web サイト訪問や買い物かごへの追加など）を考慮して、指定されたカスタム目標に含まれる最終コンバージョンイベントに対して、最も低い CPA のプレースメントを優先します。 最適化モデルをクリックベースのコンバージョンからのみ学習するか、クリックベースのコンバージョンとインプレッションベースのコンバージョンの両方から学習するかを指定できます。<br><br>オークション評価では CPA が優先されます。 費用の目標が満たされている場合、DSPは CPM の低下と CPA の低下のバランスを取ります。 | キャンペーンタイプ : パフォーマンス<br><br>ベンチマーク：最低 CPA<br><br>広告タイプ：ディスプレイ、ネイティブ、ビデオ、CTV およびユニバーサルビデオ<br><br><b>注意：</b> 参照先」[パフォーマンスキャンペーンの設定のベストプラクティス](/help/dsp/optimization/campaign-best-practices-performance.md)」を参照してください。 |
| [!UICONTROL Lowest Cost per Click] | パッケージレベルの最適化を使用して、予算割り当てでは CPC が最も少ないプレースメントが優先されます。<br><br>オークション評価では CPC が優先されます。 費用の目標が満たされている場合、DSPは CPC を下げようとすると同時に、CPM の下げを上げます。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高のクリックスルー率<br><br>広告タイプ：プリロール、ディスプレイ<br><br><b>注意：</b> この目標を使用して、可能な限り最高の CPC を達成します。 最大 CPM を保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest Cost per Completion] | パッケージレベルの最適化を使用して、予算割り当ては、完了あたりのコストが最も低いプレースメントを優先します。<br><br>オークション評価では、ビデオ完了率（VCR）が優先されます。 費用の目標が満たされている場合、DSPは完了あたりのコストを下げるように、CPM の削減と VCR の増加のバランスを取ります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高の完了率<br><br>広告タイプ：プリロールのみ |
| [!UICONTROL Lowest Cost per Engagement] | パッケージレベルの最適化を使用して、予算割り当てでは、エンゲージメント率が最も低いプレースメントが優先されます。<br><br>オークション評価では、契約率が優先順位付けされます。 費用の目標が満たされている場合、DSPは CPM の削減とエンゲージメントあたりのコストの削減のバランスを取ろうとします。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高のエンゲージメント率<br><br>広告タイプ：Mobile Interstitial のみ |
| [!UICONTROL Lowest Cost per Reach] | この目標は、特定の予算で世帯の最大範囲を達成することを試みます。 費用の目標が満たされている場合、ユニークなリーチを段階的に達成する機会によって入札が異なります。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：リーチあたりの効率的なコスト<br><br>広告タイプ：プリロール、ディスプレイ、CTV、ネイティブ、オーディオおよびユニバーサルビデオ |
| [!UICONTROL Lowest Cost per View] | 最小 CPM と同様に動作します。 パッケージレベルの最適化を使用して、予算割り当ては CPM が最も低いプレースメントを優先します。<br><br>オークション評価では CPM が優先されます。 費用の目標が満たされている場合は、DSPが CPM を徐々に下げます。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高のクリックスルー率<br><br>広告タイプ：プリロール、ディスプレイ |

| [!UICONTROL Lowest vCPM (Adobe - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては vCPM が最も低いプレースメントを優先します。<br><br>オークション評価では vCPM が優先されます。 費用の目標が満たされている場合、DSPは、CPM の低下とビューアビリティの向上のバランスをとろうとします。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高の vCPM<br><br>広告タイプ：プリロール、ディスプレイ<br><br><b>注意：</b> この目標を使用して、可能な限り最高の vCPM を達成します。<br><br>最大 CPM を保証するには、プレースメントの最大入札額として使用します。 |
| [!UICONTROL Lowest vCPM (IAS - MRC)] | パッケージレベルの最適化を使用して、予算割り当ては vCPM が最も低いプレースメントを優先します。<br><br>オークション評価では vCPM が優先されます。 費用の目標が満たされている場合、DSPは、CPM の低下とビューアビリティの向上のバランスをとろうとします。 | キャンペーンタイプ：ブランディング<br><br>ベンチマーク：効率的な CPM と最高の vCPM<br><br>広告タイプ：プリロール、ディスプレイ<br><br><b>注意：</b> この目標を使用して、可能な限り最高の vCPM を達成します。<br><br>最大 CPM を保証するには、プレースメントの最大入札額として使用します。<br><br>この設定は、IAS のサードパーティデータがアルゴリズムに通知している場合に最適です。 この目標は、キャンペーンに対して IAS 統合を有効にした場合にのみ使用します。 |
